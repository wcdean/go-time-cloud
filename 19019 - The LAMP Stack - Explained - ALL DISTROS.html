Conceptual guide to LAMP Stack and its components, LAMP

The LAMP stack explained - ALL DISTROS

Difficulty: 3

Time: 30 minutes

This article is an overview of the LAMP stack. LAMP is a widely-used, open source web platform used to run dynamic websites and servers. Inspired by the names of the components (Linux®, Apache™, MySQL™, and PHP™), the acronym LAMP represents a generic software model. Being completely free and open source has made LAMP a popular choice with developers. LAMP stack combined with all Linux distributions has become a development standard today.

The LAMP bundle is sometimes combined with other free, open source packages, such as:

  • Netsniff (for security testing and hardening)
  • Snort (for intrusion detection and prevention)
  • RRDtool (for diagrams)
  • Nagios, Collectd (for monitoring)

Original LAMP has spawned many other related acronyms that focus on providing feature rich websites, such as LAPP, MAMP, and BAMP.

The components

If you simply explore the acronym letter by letter, you get the following components:

  • Linux
  • Apache web server
  • MySQL database
  • Perl, Python, or PHP

Each component is powerful in itself, and is an example of Free and Open Source Software (FOSS). The FOSS approach has the following benefits:

  • FOSS is freely available to download, making it easy to distribute.
  • FOSS licenses are open, so there are fewer restrictions on the deployment of applications.
  • Users have access to the source, so it is easy to fix faults and improve the software.

Linux

Linux is a Unix-like computer operating system developed under FOSS development and distribution. Being an operating system, it is capable of doing much more than a web application platform. Linux is built on a collection of software based on the Linux kernel and a package management system that gives complete LAMP setups. You can customize Linux kernel to get the best performance out of it. It has been developed for a very wide range of platforms, from PCs to servers, to mainframe hardware and embedded systems. Thus, older hardware, which is no longer supported by Original Equipment Manufacturer (OEM), can now be used to run Linux.

Despite all complexities, the distribution model of Linux, which packages different components, makes installation and development of Linux server very smooth. You can get distributions specially designed to develop and deploy LAMP, to further simplify the LAMP technology.

Apache

Apache HTTP server is the most popular web server. In June 2013, Netcraft concluded that Apache serves 54.2 % of all the active websites and 53.3% of the top servers. In June 2014, it was found that Apache serves 52.27% of active websites, followed by NGINX with 14.36%.

Apache Software Solutions maintains and develops Apache HTTP server. It is an open-source software released under an Apache license. Many other features are supported and developed as compiled modules, to extend the core functionality of Apache, such as server-side programming language support and authentication schemes.

Apache project is a generic name for many high-quality, open source software, but its original project was HTTPS server. Apache supports basic file serving CGI to provide interactive application or loadable modules.

Apache is easy to install, configure, and maintain, and requires less manual effort.

MySQL

MYSQL is an example of a simple, but powerful application. It can be installed without any complex configuration, in the same way as Apache and Linux are installed. The default configuration already makes it a very powerful relational database, but if you need to optimize it further, the configuration can be customized.

MySQL is a multi-threaded, multi-user, SQL database management system (DBMS). Its source code is available under GNU general public license and a variety of proprietary agreements.

MariaDB is a fork of MySQL, which is community-developed. PostgreSQL is an ACID-compliant relational database, not related to MySQL.

MySQL supports relational tables, and SQL makes it easy to develop enterprise level databases. MySQL is particularly more efficient with more number of queries, but less updates.

Applications with high rate of transactions, use InnoDB.

Perl, Python, or PHP

Perl, Python, and PHP are interpreted languages, and therefore you can run the application without compiling it. CGI interface was built into early webservers, as a result, all these languages were commonly used then. CGI made it possible to communicate between a client browser and an external application, which gave rise to dynamic content. Interpreted languages made it very easy to develop such applications.

Early web developers mostly used Perl when dealing with text elements. Perl made it easy to process data via a CGI interface and to manipulate text files. Later, the mod_perl extended the use of Perl as a dynamic language.

PHP was originally a suite of Perl script to create dynamic pages with CGI interface. Later, it was developed as a separate language, but it still holds much of the language semantics of Perl. For some programmers, PHP is more logical than Perl, and the way that it embeds with HTML makes it easy to read and code. PHP is a server-side scripting language, but is also used in general purpose programming.

Python is a general purpose high-level programming language. It supports object oriented, imperative, and functional programming paradigm. It features automatic memory management, standard library, and a strict use of white space. It is also used as a scripting language.

LAMP as a whole and other Linux-based solutions

There are some commonly used Linux-based solutions:
  • MEAN : A free, open-source JavaScript software stack to develop dynamic web applications. It is a combination of MogoDB,Expression.js, and Angular.js.
  • NGINX : A free, open-source, high-performance HTTP server and reverse proxy. It can work as IMAP/POP3 proxy server too.
  • LEMP : Stack is a group of open-source software to get web servers up and running. The acronym stands for Linux, NGINX, MySQL, and PHP.
  • LAPP : Considered to be an alternative to the LAMP stack, it includes Linux, Apache, PostgreSQL and PHP, and Python and Perl.

As we have seen in this article, by combining a webserver, a dynamic component, and a database, we can easily create database-driven dynamic websites. This is the power of LAMP. It is often considered as the best free and open source solution to compete with other commercial solutions, such as IIS, Windows, and .NET. It is a popular solution, not only in terms of cost, but also performance and optimization.
