Setting up Rails application with Unicorn and NGINX in Ubuntu, Ruby, Rails,NGINX

Deploy Rails app with Unicorn and NGINX – Ubuntu

Difficulty: 2

Time: 30 minutes

In this article, we will be configuring a multi-layer deployment installation to host Rails-based Ruby web applications. To do this, we will use the powerful, flexible, and extremely successful Unicorn application server running behind NGINX.

We assume that you are logged in as a sudo user.

Prerequisites

We assume that you have gone through the articles: Install Ruby on Rails with rbenv - Ubuntu and Use PostgreSQL with your Ruby on Rails Application - Ubuntu.

Prepare the deployment server

  1. Install the bundle of development tools, and then update the operating system using the following commands:
    sudo apt-get install build-essential
    sudo apt-get update
  2. Install NGINX.
    sudo apt-get install nginx
  3. Start the NGINX service.
    sudo service nginx start
  4. Install Unicorn.
    gem install unicorn

Set up a Rails Application for deployment

  1. Go to home directory and create a Rails Application in this directory.
    cd /home/logged_in_username
    rails new helloworld -d postgresql
  2. Go to the application directory.
    cd helloworld/
  3. Create a user for the production database.
    sudo -u postgres createuser -s username
  4. To set database user password, run the following commands:
    sudo -u postgres psql
    \password username
  5. Enter the new password, and then log out using the following command:
    \q

Set up a database connection

  1. To set up a database connection, you need to move to your application directory.
    cd /home/logged_in_username/helloworld
    sudo vim config/database.yml
  2. Add the following line in the default section of this file: 
    host: localhost
  3. Locate the production section at the bottom of the file, and update the highlighted section with appropriate values: 
    username: username
password: <%= ENV['helloworld_DATABASE_PASSWORD'] %>

    If our production username and the database user are not the same, then make the required changes to make them the same.

  4. Save and exit the file by pressing the Esc key, and then typing :wq, followed by the Enter key.

Set up rbenv plug-ins

  1. Install the plug in for rbenv-vars.
    cd
    cd .rbenv/plugins
    git clone https://github.com/sstephenson/rbenv-vars.git
  2. Set environments variable. To generate a secret key, run the following command:
    cd /home/logged_in_username/helloworld
    sudo apt-get install libpq-dev
    bundle install
    rake secret

    Copy the generated secret key into any text editor (like notepad) for later reference.

  3. Open the following file:
    sudo vim .rbenv-vars
    Add the following line in this file: 
    SECRET_KEY_BASE=your_generated_secret
Your_application_DATABASE_PASSWORD=prod_db_pass
  4. Save and exit the file by pressing the Esc key, and then typing :wq, followed by the Enter key.
  5. Export the secret key and the database password.
    rbenv vars

Create the databases

  1. Create the sample databases for the Rails application and generate a scaffold controller to manage the database.
    RAILS_ENV=production rake db:create
    RAILS_ENV=development rake db:create
    rails generate scaffold Task title:string note:text
  2. Update the production database and run your application from the application directory.
    RAILS_ENV=production rake db:migrate
    RAILS_ENV=development rake db:migrate
    rails server
  3. Create a directory to hold the PID files.
    mkdir pids
  4. Enter the application directory and run the following command:
    rails s

    To terminate a user, press Ctrl+C.

  5. For binding to web browser, run the following command:
    rails server --binding=Public IP Address of your server
  6. You can now access it on any browser by visiting: http://Public IP Address of your server:3000/tasks.

Configure Unicorn and NGINX

Configure Unicorn

  1. To configure Unicorn, move to the Rails application directory.
    cd /home/logged_in_username/helloworld
  2. Create a blank unicorn.rb document.
    sudo vim config/unicorn.rb
  3. Add the following code in this file: 
    # set path to application
	app_dir = File.expand_path("../..", __FILE__)
	shared_dir = "#{Application directory}/shared"
	working_directory Application directory


	# Set unicorn options
	worker_processes 2
	preload_app true
	timeout 30

	# Set up socket location
	listen "#{shared_dir}/sockets/unicorn.sock", :backlog => 64

	# Logging
	stderr_path "#{shared_dir}/log/unicorn.stderr.log"
	stdout_path "#{shared_dir}/log/unicorn.stdout.log"

	# Set master PID location
	pid "#{shared_dir}/pids/unicorn.pid"
  4. Save and exit the file by pressing the Esc key, and then typing :wq, followed by the Enter key.

Connect Unicorn with Rails Application

  1. Create the directories that were referred in the configuration file.
    sudo mkdir -p shared/pids shared/sockets shared/log
  2. Create a script for Unicorn init.
    sudo vi /etc/init.d/helloworld_unicorn
  3. Add the following script code in the file: 
    	#!/bin/sh

	### BEGIN INIT INFO
	# Provides:          unicorn
	# Required-Start:    $all
	# Required-Stop:     $all
	# Default-Start:     2 3 4 5
	# Default-Stop:      0 1 6
	# Short-Description: starts the unicorn app server
	# Description:       starts unicorn using start-stop-daemon
	### END INIT INFO

	set -e

	USAGE="Usage: $0 &lt;start|stop|restart|upgrade|rotate|force-	stop>"

	# app settings
	USER="username"
	APP_NAME="helloworld"
	APP_ROOT="/home/logged_in_username/$APP_NAME"
	ENV="production"

	# environment settings
	PATH="/home/$USER/.rbenv/shims:/home/$USER/.rbenv/bin:$PATH"
	CMD="cd $APP_ROOT && bundle exec unicorn -c config/unicorn.rb -	E 	$ENV -D"
	PID="$APP_ROOT/shared/pids/unicorn.pid"
	OLD_PID="$PID.oldbin"

	# make sure the app exists
	cd $APP_ROOT || exit 1

	sig () {
	  test -s "$PID" && kill -$1 `cat $PID`
	}

	oldsig () {
	  test -s $OLD_PID && kill -$1 `cat $OLD_PID`
	}

	case $1 in
  	start)
   	 sig 0 && echo >&2 "Already running" && exit 0
   	 echo "Starting $APP_NAME"
   	 su - $USER -c "$CMD"
  	  ;;
 	 stop)
  	  echo "Stopping $APP_NAME"
  	  sig QUIT && exit 0
 	   echo >&2 "Not running"
  	  ;;
 	 force-stop)
  	  echo "Force stopping $APP_NAME"
  	  sig TERM && exit 0
	    echo >&2 "Not running"
  	  ;;
  	restart|reload|upgrade)
  	  sig USR2 && echo "reloaded $APP_NAME" && exit 0
  	  echo >&2 "Couldn't reload, starting '$CMD' instead"
   	 $CMD
  	  ;;
  	rotate)
  	  sig USR1 && echo rotated logs OK && exit 0
  	  echo >&2 "Couldn't rotate logs" && exit 1
  	  ;;
 	 *)
  	  echo >&2 $USAGE
  	  exit 1
 	   ;;
	esac

    This script will allow to execute helloworld_unicorn that will run and stop both the unicorn and Rails application.

  4. Save and exit the file by pressing the Esc key, and then typing :wq, followed by the Enter key.
  5. Update the script's permissions and enable Unicorn to start on boot.
    sudo chmod 755 -R /etc/init.d/helloworld_unicorn
    sudo update-rc.d helloworld_unicorn defaults
  6. Start Unicorn.
    sudo service helloworld_unicorn start

Configure NGINX

  1. Configure NGINX and enable it to communicate with Unicorn. Edit the default configuration file: default.conf and leave nginx.conf.
    sudo vim /etc/nginx/sites-available/default

    Replace the files contents with the following lines:

       upstream app {
    		# Path to Unicorn SOCK file, as defined previously
   		server 	unix:/home/Logged_in_username/Helloworld/shared/sockets/unicorn.sock 	fail_timeout=0;
	}

	server {
   	 listen 80;
   	 server_name localhost;

  	  root /home/Logged_in_username/Helloworld/public;

   	 try_files $uri/index.html $uri @app;

   	 location @app {
        	 proxy_pass http://app;
       		 proxy_set_header X-Forwarded-For 			  		
			 $proxy_add_x_forwarded_for;
        	 proxy_set_header Host $http_host;
        	 proxy_redirect off;
    	}

   	 error_page 500 502 503 504 /500.html;
        client_max_body_size 4G;
    	 keepalive_timeout 10;
	}
  2. Save and exit the file by pressing the Esc key, and then typing :wq, followed by the Enter key.
  3. Restart the NGINX service.
    sudo service nginx restart
  4. For binding to web browser, run the following command:
    rails server --binding=Public IP Address of your server
  5. Test your server's IP address or the domain name associated to it. Go to http://Your server's IP address:3000/tasks.

Conclusion

By using all the above steps, you will be able to deploy our Rails application with Unicorn and NGINX on Ubuntu.
