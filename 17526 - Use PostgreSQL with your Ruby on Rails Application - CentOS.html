Setting up a Rails Application with PostgreSQL in CentOS 7, PostgreSQL, Ruby, Rails.

Use PostgreSQL with Ruby on Rails Application - CentOS 7

Difficulty: 2

Time: 30 mins

This article will assist you to create a PostgreSQL server that will be used as a database layer for scaling Ruby on Rails (RoR) application. It describes how you can use PostgreSQL for Rails application. These tasks assume that you have sudo privileges and that Ruby on Rails is installed.

Set up a PostgreSQL database

  1. Download the latest EPEL repositories.
    sudo wget http://dl.fedoraproject.org/pub/epel/7/x86_64/e/epel-release-7-5.noarch.rpm
    sudo rpm -ivh epel-release-7-5.noarch.rpm
  2. Set up a PostgreSQL server, and add its development-related libraries.
    sudo yum install postgresql-server postgresql-contrib postgresql-devel
  3. Initialize a new PostgreSQL database.
    sudo postgresql-setup initdb
  4. Open the pg_hba.conf file, so that you can modify the host-based authentication of PostgreSQL. It does not allow for password authentication by default.
    sudo vim /var/lib/pgsql/data/pg_hba.conf
  5. Search for the following lines at the end of the file: 
    host    all             all             127.0.0.1/32            ident
host    all             all             ::1/128                 ident
  6. Change ident to md5 so that it looks like this: 
    host    all             all             127.0.0.1/32            md5
host    all             all             ::1/128                 md5
    Save and exit the file.
  7. Run and enable the PostgreSQL service.
    sudo systemctl start postgresql
    sudo systemctl enable postgresql

Create a database user

  1. Log in as postgres system user.
    sudo su - postgres
  2. Create a PostgreSQL super user.
    createuser -s username
  3. Set a password for the database user.
    \password username
  4. Enter the new password.
  5. Close the session.
    \q
  6. Logout form postgres user.
    logout

Set up a Rails application

  1. Move to the home directory to create a new Rails application, and set PostgreSQL as the database.
    cd ~
    rails new your_application -d postgresql
  2. Move to the application directory.
    cd your_application
  3. Open the following file:
    sudo vim config/database.yml
  4. Look for pool: 5 in the file, and add the following lines beneath it. Enter the username and password for the postgres user. 
          
  pool: 5  
  host: localhost
  username: username
  password: username_password

    When adding this code, you need to ensure proper alignment with the code "pool: 5". Also spacing with in the code added must be maintained as shown above.

  5. Save and exit the file.

Create a database for the application

  • Create the application's databases in the application directory.
    rake db:create
    Two databases in the PostgreSQL server are created. For example, if the application's name is HelloWorld, the command creates databases named HelloWorld _development and HelloWorld _test.

Test the configuration

  • Test the Rails application.
    rails server
    You can run the Rails application on your localhost on port 3000. You can also access the application from a web browser, if it is located on a remote server, by assigning a public IP address.
    rails server --binding=server_public_IP
    On your web browser navigate to http://server_public_IP:3000 and you will see a page with "Welcome aboard. You’re riding Ruby on Rails!" on it.
