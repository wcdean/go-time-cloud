Install WordPress on your LAMP server - Ubuntu

Difficulty: 1
Time: 15 minutes

WordPress has become much more than its humble beginnings in 2003 as a self-hosted blogging tool; it is the Internet's leading content management system (CMS). This open-source CMS powers all types of sites, including major brands such as The New Yorker, Variety, TechCrunch, and Best Buy. With a MySQL and PHP architecture, WordPress is easy to install and has endless options for website configuration thanks to tens of thousands of plugins and themes from its rich and active community.

This tutorial will walk you through setting up WordPress on a Ubuntu™ server with PHP and MySQL.

Prerequisites

  1. A Ubuntu server with a LAMP stack (Linux, Apache, MySQL, PHP).
  2. Log in to your server with a user that has sudo privileges. If you have a GoDaddy Cloud server, you will already have one by default.
  3. (OPTIONAL) Point a domain or subdomain to your server. If you don't want to use a real domain yet, you can edit your hosts file to test your configuration. If you don't want to use a domain with your WordPress site just yet, you can instead simply use your server IP address when testing your site.

Create a MySQL database and user

  1. Log into your MySQL admin account.
    sudo mysql -u root -p
  2. At the prompt, enter your MySQL root password. Use the root password you set up when you installed MySQL.

    If you do not remember your root MySQL password, see our guide to reset your root MySQL password.

  3. Create a new MySQL database.
    CREATE DATABASE wordpress;
  4. Create a new MySQL user and password.
    CREATE USER wpuser@localhost;
  5. Set the password for this new user. Make sure to use a safe passwword.
    SET PASSWORD FOR wpuser@localhost= PASSWORD("wppassword");
  6. Grant all privileges to this new user.
    GRANT ALL PRIVILEGES ON wordpress.* TO wpuser@localhost IDENTIFIED BY 'wppassword';
  7. Confirm that the changes to this user are properly applied.
    SHOW GRANTS FOR 'wpuser'@'localhost';

    If the changes worked, you should see an output that looks something like this:
    GRANT USAGE ON *.* TO 'wpuser'@'localhost' IDENTIFIED BY PASSWORD '*977FDAC11B3F74324912DA4F9D489A3E6D59B281'
    GRANT ALL PRIVILEGES ON `wordpress`.* TO 'wpuser'@'localhost'
  8. If the changes did not apply properly, refresh the MySQL cache and try again.
    FLUSH PRIVILEGES;
  9. Once you confirm that the changes worked, exit MySQL.
    exit

Download WordPress

You are ready to download and install WordPress.

  1. Navigate to your Apache Document Root.
    cd /var/www/html

    This guide uses the default DocumentRoot for Apache on Ubuntu. If you already set up an Apache VirtualHost on your Ubuntu server, use that directory instead of /var/www/html.

  2. Give this directory the correct permissions settings.
    sudo chmod -R 755 /var/www/html
  3. Download the latest version of WordPress.
    sudo wget http://wordpress.org/latest.tar.gz
  4. Extract the WordPress files.
    sudo tar xvzf latest.tar.gz
  5. Move these files into the current directory.
    sudo mv ./wordpress/* .
  6. Give the Apache user ownership of these files.
    sudo chown -R www-data:www-data /var/www/html
  7. Remove the unused wordpress directory and the install package.
    sudo rmdir wordpress
    sudo rm latest.tar.gz

Configure and Install WordPress

You are now ready to start setting up your WordPress files for installation. You will start by connecting your WordPress files to the MySQL database you created earlier in this guide. This section of the guide will be done in your web browser.

  1. Enter your server's domain name or public IP address in your web browser.
    http://your_domain_name_or_IP
  2. Choose your language, then click Continue.
  3. You will see a screen asking for your MySQL database credentials. Click Let's go!.
  4. Fill in the fields for your database information. These are the MySQL credentials you created earlier. For the hostname, use localhost. For the Table Prefix, you can keep the default of wp_, but it is often a better security practice to use a randomly generated prefix such as wQ2z_.
  5. Once all the database information is entered, click Submit. This will create your wp-config.php file, which contains your database connection strings.
  6. Click Run the install to begin installing WordPress.
  7. Fill out the information for your Site Title, your WordPress administrator username and password, and the email address for that user.
  8. Click Install WordPress to complete the installation.
  9. You can now log in by clicking Log in, or by navigating to http://your_domain_name_or_IP/wp-admin in your browser.

Next steps

You have installed WordPress on your server and you are now ready to start building with WordPress. For additional configruation options, or for information on how to start using WordPress, check out the official WordPress documentation.
