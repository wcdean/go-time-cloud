Set up WordPress domain mapping in Apache - Debian

Difficulty: 1
Time: 10 minutes

The WordPress MU Domain Mapping plug-in allows you to point multiple domains to one WordPress account. This means that you can make a network site into its own unique, stand-alone domain. In our tutorial, we'll map a subdomain to an independent domain. For instance, coolexample.net would redirect to product.coolexample.com, a subdomain of your primary site, coolexample.com.

Prerequisites

You should have WordPress multisite installed.

Download the plug-in

  1. First, install unzip, if you haven't already.
    sudo apt-get install unzip
    If prompted, enter y to continue installation.
  2. Download the WordPress Domain mapping plug-in.
    cd ~
    wget http://downloads.wordpress.org/plugin/wordpress-mu-domain-mapping.latest-stable.zip
  3. Unzip the plug-in.
    unzip wordpress-mu-domain-mapping.latest-stable.zip
  4. Move the files to the WordPress /wp-content/plugins/directory directory.
    sudo mv wordpress-mu-domain-mapping /var/www/wp-content/plugins/
  5. Copy the sunrise.php into your WordPress document root.
    sudo cp /var/www/wp-content/plugins/wordpress-mu-domain-mapping/sunrise.php /var/www/wp-content/

Edit your wp-config file

  1. Let's go to your WordPress document root.
    cd /var/www
  2. Open your wp-config file.
    sudo vim wp-config.php
  3. Add this line before /* That's all, stop editing! Happy blogging. */.
    define('SUNRISE', 'on');
  4. Save and close.
  5. Restart Apache.
    sudo service apache2 restart

Activate Domain Mapping

  1. Return to your wp-admin window.
  2. In the top navigation bar, point to My Sites, and from the drop-down menu, select Network Admin, then Plugins.

    You will see three plug-ins. The WordPress MU Domain Mapping plug-in should be the third item. Click this checkbox.
  3. Under this plug-in, click Network Activate. The Plugins dialog will highlight, indicating it's activated.

  4. In the menu bar on the left-hand side, point to Settings, and then select Domain Mapping from the menu that appears. Find the Domain Options section.

  5. Clear the Remote Login check box.
  6. Select the Permanent Redirect check box.
  7. Clear the Redirect administration pages to site's original domain check box.
  8. Click Save.

Your external domain will now redirect to the subdomain you have defined. For example, coolexample.net redirects to product.coolexample.com. This includes the administration pages /wp-admin.

In the next step, we map a domain name to each site based on its site ID. There's more than one way to find the ID of a site, but for easier administration, we will create a simple WordPress Multi-use plug-in that displays an additional ID column on the Sites page.

Map a domain name to each site based on site ID

  1. In your terminal, create a directory to house the plug-in.
    cd ~
    sudo mkdir /var/www/wp-content/mu-plugins
  2. Create a new file.
    sudo vim /var/www/wp-content/mu-plugins/wpms_blogid.php
    It is a new file, so it will be blank.
  3. Copy this whole code block and paste it into your file. 
    <?php
add_filter( 'wpmu_blogs_columns', 'do_get_id' );
add_action( 'manage_sites_custom_column', 'do_add_columns', 10, 2 );
add_action( 'manage_blogs_custom_column', 'do_add_columns', 10, 2 );

function do_add_columns( $column_name, $blog_id ) {
    if ( 'blog_id' === $column_name )
        echo $blog_id;
    return $column_name;
}

function do_get_id( $columns ) {
    $columns['blog_id'] = 'ID';
    return $columns;
}     
?>
  4. Save and close.
  5. Back in the WP-Admin browser window, the Sites > All Sites section should now show an additional ID column. Make note of the ID for your subdomain that you'll be mapping.

Map your domains

  1. Make a note of the ID value for each site.
  2. Under Settings, click Domains.
  3. On the Domain Mapping page, under New Domain, enter the Site ID, followed by the external domain for the site.

  4. Click Save.

Next steps

Congratulations! Each site that you've defined with its own domain name can now be accessed that way. You can test it by visiting one of those domains. You will see the URL redirect to the subdomain of your primary site.

Now each site can be maintained separately through its own WordPress admin panel. Just append /wp-admin/ to the URL, such as coolexample.net/wp-admin/.
