Managing Ruby installations and environments using RVM, Ruby, Rails

Use RVM to manage Ruby installations and environments - ALL DISTROS

Difficulty: 2
Time: 3 hours

Ruby® is a very popular object-oriented, general-purpose programming language that has Ruby on Rails™ or simply Rails, which is written in Ruby. Rails are an open source Ruby framework for developing database-backed web applications.

Generally, there is a need for different versions of Ruby installed in the same server for different projects. We can achieve this with the help of Ruby Version Manager (RVM).

In this article, we will be discussing on managing Ruby installations using RVM. We will continue assuming that you have already installed RVM on server.

If you have not yet installed RVM, refer the following articles:

  • Install Ruby on Rails using RVM - Debian

RVM Command

After RVM is installed in server, you will have rvm command to manage the Ruby installations.

  • rvm command_options command ruby_to_act_on
    For example:
    rvm install ruby

General commands and their purposes

TASK COMMAND
Get the list of different command options available rvm help
Get details about a particular command rvm help command
Find the installed packages in RVM rvm info
Update RVM rvm get stable
Enable Tab completion in .bashrc file [[ -r $rvm_path/scripts/completion ]] && . $rvm_path/scripts/completion

Commands for install, uninstall, and version management

TASK COMMAND
Get the list of available Rubies to install rvm list known
Install a selected Ruby version rvm install ruby_version
Get the list of installed Rubies rvm list
Switch between the multiple versions of Ruby installed rvm use other_ruby_version
Set a particular Ruby version as default rvm command ruby_version --default
Switch to a default Ruby rvm default
Uninstall a particular Ruby version rvm uninstall ruby_version

Manage gemsets with RVM

Gems are packages, which contains package information along with files to install. The gem command is used to build and install gem packages. A gemset is a set of gems collected together.

When you install Ruby using RVM, it comes with two gemsets:

  1. Default: This gemset is used when no other gemset is specified.
  2. Global: This gemset is inherited by every other gemset that is used.

Commands for managing Gemsets

TASK COMMAND
List the available gemsets for a current Ruby rvm gemset list
List gemsets for more than one Ruby installation rvm gemset list_all
Create a gemset rvm gemset create your_gemset_project
Copy a current gemset to a new gemset rvm gemset copy default your_gemset_project
Use a particular gemset rvm gemset use your_gemset_project
Set default version for Ruby, when you have more than one gemset installed rvm --default use ruby_version@your_gemset_project
Delete a gemset rvm gemset delete your_gemset_project
Create project specific configurations to specify which Ruby version and gemset to use in a .rvmrc file rvm ruby_version@gemset

Configurations

  • RVM's default configuration file is:
    ~/.rvm/config/db

    If RVM is upgraded, then the customizations made to the configuration file will be lost forever. Therefore, make custom configurations in '~/.rvm/user/db', so that the settings will be retained.

Conclusion

In this article, you learned the steps involved in managing Ruby using Ruby Version Manager. These commands can be used on any Linux distribution.
