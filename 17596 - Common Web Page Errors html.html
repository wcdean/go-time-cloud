Common Web page errors

Difficulty: 1
Time: 10 minutes

Introduction

HTTP status codes are three-digit numbers that provide Web browsers with information about the page's status.

Here's a quick guide to help you understand the most common error codes, with suggestions for what to do to fix the error.

List of Common Error Codes

Error Code Description
400 — Bad Request This error is the frequently due to programming problems. You should talk to your developer or software provider for help resolving this issue.
401 — Authentication Required This error displays if the page requires a user name and password to access it. If you try to access it without it, you get this error.
403 — Forbidden This errors displays when somebody tries to access a directory, file, or script without appropriate permissions.
404 — Not Found This error displays if the URL doesn't exist. The cause can be anything from invalid URLs, missing files, or redirects to URLs that no longer exist.
500 — Internal Server Error This is a very general error that means there's a problem with the website displaying, but the details aren't readily available. To find out what causes the issue, review your server's error logs.

Conclusion

These are the main error codes that you could see if you experience web page errors. For more information, check out the list of status code definitions at W3.org: http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html
