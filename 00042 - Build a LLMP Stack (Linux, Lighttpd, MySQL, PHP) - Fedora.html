Install Lighttpd, MySQL, and PHP on a server running Fedora to serve web applications.

Build a LLMP (Linux, Lighttpd, MySQL, PHP) stack on a server running Fedora

Difficulty: 3
Time: 20 minutes

LLMP is an acronym used for the software suite that is used by developers and administrators to serve web applications quickly on a stable platform. In this article, you will learn how to build a LLMP stack on Fedora 20 and 21.

Prerequisites

You must have completed the initial server setup for Fedora. Linux is installed with Fedora.

Install Lighttpd

  1. Before you begin any installation, make sure that you install all the updates.
    sudo yum update
  2. Install Lighttpd.
    sudo yum install lighttpd
  3. Start the service.
    sudo systemctl start lighttpd.service
  4. To verify that Lighttpd is installed, type this URL in a browser: http://your server address. You will see a Lighttpd test page. If you do not know your IP address, you can look it up.

Install MySQL

  1. Install MySQL.
    sudo yum install mysql mysql-server
  2. After the installation is complete, start the service.
    sudo service mysqld start
  3. Run the following security script:
    sudo /usr/bin/mysql_secure_installation
  4. You will be asked for the root password. Because you didn't set it earlier, press Enter to set a password now.
  5. Type Y to set the root password.
  6. Enter and confirm the new password.
  7. You will be asked more questions as part of security configuration. It is a best practice to respond Y to these system prompts.

Install PHP

  1. To install PHP, open the terminal and type:
    sudo yum install php-fpm lighttpd-fastcgi php-mysql
    Enter Y to install.
  2. Add lighttpd to the user and the group in the www.conf
    sudo vim /etc/php-fpm.d/www.conf
    Find the following block : 
    ; Unix user/group of processes
; Note: The user is mandatory. If the group is not set, the default user's group
;       will be used.
; RPM: apache Choosed to be able to access some dir as httpd
user = apache
; RPM: Keep a group allowed to write in log dir.
group = apache
    3. Change user=apache to user=lighttpd and group=apache to group=lighttpd.
  3. Save and exit the file.
  4. This step is recommended, even though it's optional. Create a system startup link for PHP-FPM to start every time the system boots up.
    sudo chkconfig --levels 235 php-fpm on
  5. Start the service.
    sudo systemctl start php-fpm.service

Configure Lighttpd with PHP

  1. Open the php.ini file.
    sudo vim /etc/php.ini
  2. Uncomment the following line :
    cgi.fix_pathinfo=1
  3. Save and exit the file.
  4. Edit the modules.conf file.
    sudo vim /etc/lighttpd/modules.conf
    Uncomment the following line :
    include "conf.d/fastcgi.conf"
  5. Save and exit the file.
  6. Edit the fastcgi.conf file.
    sudo vim /etc/lighttpd/conf.d/fastcgi.conf
    Enter the following block of code just after the server.modules += ( "mod_fastcgi" ) line and before the PHP Example. 
    fastcgi.server += ( ".php" =>
        ((
                "host" => "127.0.0.1",
                "port" => "9000",
                "broken-scriptfilename" => "enable"
        ))
)
  7. Save and exit the file.
  8. Restart Lighttpd.
    sudo systemctl restart lighttpd.service

Test PHP processing on Lighttpd

  1. Create a new PHP file under the /var/www/lighttpd directory.
    sudo vim /var/www/lighttpd/info.php
  2. When the file opens, type in the following code:
    <?php
    phpinfo();
    ?>
  3. Save and exit the file.
  4. To verify that it works, type this URL in your browser: http://your IP address/info.php. If it works, a page displays the PHP version, extensions, build date, and other information.

Next steps

Congratulations! You have successfully built a LLMP stack on your server.
