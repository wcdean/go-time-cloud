What are MySQL and MariaDB?

MySQL and MariaDB are relational databases. These tools are used to handle large volumes of data for different kinds of application programs. Both tools use SQL query language to retrieve the information from different tables and databases. Both MySQL and MariaDB support Virtual Private Server (VPS) or cloud to manage your data.

Login to MySQL/MariaDB

Make sure that MySQL/MariaDB is installed on machine as well as database already exist.
  • Login to MySQL/MariaDB terminal.
    mysql -u mysql_username -p
    Enter MySql password when prompted

Create sample database

  • Create database using the following command if database is not exist.
    CREATE DATABASE database_name;

Create sample table

  • Create database using the following command if database is not exist.
    USE database_name;
    CREATE TABLE datatable (selected_fied1 VARCHAR(30) NOT NULL,selected_fied2 VARCHAR(30) NOT NULL);

Fetch information

  • You can tell the database server to fetch information from the datatables table by using the following query:
    SELECT selected_fied1 FROM datatables WHERE search_crietaria;

Select data source

  1. To use the default mysql database for your queries, start with this command:
    USE mysql;
    The database will confirm with the following message:
    Database changed
  2. To view the list of tables within MySQL database, and to confirm the data sources, use the following query:
    SHOW TABLES;

    There is a semicolon after every command, which indicates that the engine query ends there.

    Ubuntu will return only twenty-four results for a MySQL installation.

  3. To view the column headers for a specific table, such as user table, and to get an idea of which categories you can select from, use this command:
    SHOW COLUMNS FROM user;

Create a basic query

You can use the SELECT query to retrieve data for specific fields. The following query will return results for the three fields included in the SELECT query. Separate fields by a comma.

  • You can select specific information about a user with the following query:
    SELECT user,password,host FROM user;

Use a wildcard in queries

You can use an asterisk (*) as a wildcard to query all the information from a specified table.

  • You can select all information about a user with the following query:
    SELECT * FROM user;

Filter results

You can apply a filter to the results that meet a specific criteria by using the where keyword.

  1. You can query for user, password, and host, and then filter for the user named root.
    SELECT user,password,host FROM user WHERE user = "root";
  2. You can apply a filter for different fields. In this example, the results will return user, password, and host from the user table, which has a value named localhost for host.
    SELECT user,password,host FROM user WHERE host = "localhost";

Compare values with the Like operator

The Like operator compares the value from the right side of the expression.

  1. The following query will fetch user, password, and host fields from user table, where the criteria host is LIKE l%.
    SELECT user,password,host FROM user WHERE host LIKE "l%";
    This will result in an output with all the respective fields starting with letter "l".
  2. You can also apply and or or for equality comparison. The following query will match each record and return result with filter condition that matches the criteria, such as host starting from either l or 127.
    SELECT user,password,host FROM user WHERE host LIKE "l%" or host LIKE "127%";
