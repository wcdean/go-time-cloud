Install and configure a LEMP stack for a CentOS 7 server.

Build a LEMP stack (Linux, NGINX, MySQL, PHP) - CentOS 7

Difficulty: 3
Time: 20 minutes

LEMP is an unsual acronym, because it really should be LNMP (Linux™, NGINX®, MySQL®, PHP™). The "E" stands for NGINX (Engine X) and is probably used because of the wide acceptance of LAMP to refer to a stack built on the Apache™ operating system.
Installing a LEMP stack creates a platform for your cloud server that includes:

  • An operating system (Linux)
  • A web server (NGINX)
  • A database (MySQL)
  • An application server (PHP)

You can build a fully functioning web site and/or use it as a platform for other applications.

Install MySQL

  1. Get the local install for CentOS 07.
    sudo yum localinstall http://dev.mysql.com/get/mysql-community-release-el7-5.noarch.rpm
  2. Install MySQL.
    sudo yum install mysql-community-server

    There may be a prompt asking you if it is okay to install a key. Answer y.

  3. Activate MySQL.
    sudo systemctl start mysql.service
  4. Secure your database server.
    sudo /usr/bin/mysql_secure_installation

    The prompt will request the answer to a number of questions.

    The following table contains the remainder of the questions you will be asked, the recommended response, and some remarks regarding the consequences of not following the recommendation.

    Question Suggested Answer Why / Why not?
    Enter current password for root: Press Enter for none. Do not enter a password. If you do so, MySQL will not have sufficient access to system resources to function as a database server.
    Do you want to change the root password y A password is needed to secure MySQL. Enter a strong password and write it down.
    Remove anonymous users? y The anonymous user allows anyone to login without requiring an account created by an admin. A hacker attempting to compromise your database server may try logging in as this user.
    Disallow root login remotely? y It is more secure to create a non-root user with admin permissions. Answer "no," if you need more flexibility to manage your database server.
    Remove test database and access to it? y If you have a reason to keep a test database, answer "no."
    Reload privilege tables now? y Why wait?

  5. Activate MySQL.
    sudo systemctl restart mysql.service

Install and start NGINX

  1. Install NGINX.
    sudo yum install nginx
  2. Start the NGINX service.
    sudo systemctl start nginx.service

NGINX is now running as a web server. You can verify this using your browser to navigate to the IP address of your cloudserver, e.g. http://IP Address of your cloudserver.

Install and configure PHP

Install PHP

  • Run the following command:
    sudo yum install php php-mysql php-fpm

Configure the php.ini file

  1. Open the /etc/php.ini file.
  2. Find the cgi.fix_pathinfo directive. Most of the file is commented out and it will be about halfway through the file under a section called Paths and Directories.
  3. Uncomment the line and set it as follows:
    cgi.fix_pathinfo=0

Configure the www.conf file

  1. Open /etc/php-fpm.d/www.conf file.
  2. Find the listen directives and set it as follows:
    listen = /run/php-fpm/php-fpm.sock
  3. Configure www.conf listen.owner, listen.group, and listen.mode directives.
    Find the listen.owner,listen.group and listen.mode directives and modify them as follows:
    listen.owner = nginx
    listen.group = nginx
    listen.mode  = 0666
  4. Configure www.conf listen.owner, listen.group, and listen.mode directives.
    Find the owner and group directives and modify them as follows:
    user = nginx
    group = nginx
  5. Save and close.

Restart PHP

  • Run the following command to restart PHP:
    sudo systemctl restart php-fpm

Change permissions and ownership on php-fpm.sock

  1. Navigate to the /run/php-fpm directory:
    cd /run/php-fpm
  2. Modify the permissions and ownership for the php-fpm.sock file as follows:
    sudo chmod 666 php-fpm.sock
    sudo chown nginx php-fpm.sock
  3. Restart PHP.
    sudo systemctl restart php-fpm

Configure NGINX

  1. Open /etc/nginx/nginx.conf and verify that it includes the following line in the server context:
    include /etc/nginx/default.d/*.conf;
  2. Create a file, /etc/nginx/default.d/default.conf, and edit it to look like the example below. Your NGINX installation may include a commented out php location context so you may not have to do much more than remove the comments. If not, you will have to make the edits manually.In either case, edit the location context to look like this:
    index index.php index.html index.htm;

    # pass the PHP scripts to FastCGI server listening on the php-fpm socket
    location ~ \.php$ {
                       try_files $uri =404;
                       fastcgi_pass unix:/run/php-fpm/php-fpm.sock;
                       fastcgi_index index.php;
                       fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
                       include fastcgi_params;
                      } 
    As long as the fastcgi_pass in the nginx.conf file matches the listen directive in the PHP configuration file, NGINX should be able to pass requests on to PHP.
  3. Save and close.
  4. Restart NGINX.
    sudo systemctl restart nginx

If NGINX fails to restart, check your edits and make sure all of your braces ({}) are closed and that each directive ends with a semi-colon (;).

Verify PHP is running

  1. Create a test file, /usr/share/nginx/html/info.php, with the following contents:
    <?php
    phpinfo();
    ?>
  2. Open a browser and type: http://IP address or domain name of your server/info.php. You will see a page full of statistics with a PHP logo in the upper right-hand corner.
