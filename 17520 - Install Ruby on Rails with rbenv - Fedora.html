Install Ruby on Rails Using rbenv - Fedora

Difficulty: 1
Time: 30 minutes

Ruby is a pure object-oriented language like Perl and Python. Rails is a framework for web application development written in Ruby. In this article, you'll learn to install Ruby on Rails on Fedora with the help of rbenv, a lightweight Ruby version management tool.

You need sudo privileges to run the commands.

To set up the Fedora server, refer to Initial Server Setup - Fedora. Optionally, you can also install Git.

Install rbenv

  1. Install the dependencies of rbenv and Ruby.
    sudo yum install -y git-core zlib zlib-devel gcc-c++ patch readline readline-devel libyaml-devel libffi-devel openssl-devel make bzip2 autoconf automake libtool bison curl sqlite-devel
  2. Install rbenv.
    cd
    git clone git://github.com/sstephenson/rbenv.git .rbenv
  3. Add a directory to your $PATH for accessing rbenv.
    echo 'export PATH="$HOME/.rbenv/bin:$PATH"' >> ~/.bash_profile

  4. Add rbenv init to the shell for enabling the shims.

    echo 'eval "$(rbenv init -)"' >> ~/.bash_profile
    exec $SHELL
  5. Install the latest development version of Ruby-build into the ~/.rbenv/plugins/ruby-build directory.
    git clone git://github.com/sstephenson/ruby-build.git ~/.rbenv/plugins/ruby-build
  6. Add a directory to your $PATH for accessing Ruby.
    echo 'export PATH="$HOME/.rbenv/plugins/ruby-build/bin:$PATH"' >> ~/.bash_profile
  7. Execute the shell to install rbenv into your home directory. This also sets the appropriate environment variables for Ruby.
    exec $SHELL
    source ~/.bash_profile

Install Ruby

Before you start using rbenv, you need to install Ruby. Install the latest version unless another specific version is needed for any application dependencies.

  1. Check the latest version of Ruby using the following command.
    rbenv install --list

    Above command will display the list in unsorted format.

  2. Install the Ruby using the following command. In this article Ruby 2.2.2 is used for example.
    rbenv install latest_version of Ruby -k
  3. Set the default version of Ruby to be used by all the shells.
    rbenv global latest_version of Ruby
  4. Display the version of Ruby that was installed.
    ruby -v

Install RubyGems

  1. Stop the document installation by disabling it. (When you install gems, by default the documentation files install but take time.)
    echo "gem: --no-document" > ~/.gemrc
  2. Install the Bundler gem. The Bundler gem is an essential tool for managing gems when developing and running Rails applications.
    gem install bundler

Install JavaScript runtime

A few Rails features require a JavaScript runtime to execute. The JavaScript runtime can be installed in a couple of ways. We'll use nodejs.

  1. Add Fedora's Extra Packages for Enterprise Linux (EPEL) yum repository. This addition lets you install software packages that are not included in the official Fedora or Red Hat Enterprise Linux (RHEL) repositories.
    sudo wget http://dl.fedoraproject.org/pub/epel/7/x86_64/e/epel-release-7-5.noarch.rpm
    sudo rpm -ivh epel-release-7-5.noarch.rpm
  2. Install the Node.js package.
    sudo yum install nodejs

Set up Rails

  1. Install Rails.
    gem install rails
    This installation takes some time.
  2. Install Shims for all Ruby executables known to rbenv, using the subcommand rehash.
    rbenv rehash
  3. Find out the version of both Ruby and Rails.
    ruby -v
    rails -v
