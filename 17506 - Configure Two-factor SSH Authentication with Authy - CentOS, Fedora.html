Configure Two-Factor SSH authentication with Authy - CentOS, Fedora

Difficulty: 1

Time: 10 minutes

Authy provides an easier platform for setting up an authentication, as compared to two-factor authentication. Authy is very simple to install and manage. In this article, you will learn how to configure a two-factor SSH Authentication with Authy. For this, you need to install an Authy application in your smartphone. You will get this app in Google play.

Configure Authy in phone

  1. Download Authy from Google play, and then configure Authy by getting it verified on your smartphone.
  2. Build a developer account at http://dashboard.authy.com/signup.

    Make sure that you use the same mobile number that you used in the Authy application.

  3. Verify your account by completing the verification process sent to your registered email account. You will get a token on your smartphone.

Create an application

  1. Log in to your developer's account at http://dashboard.authy.com.

  2. Enter the Application Name, and then click Create App to create an application.

  3. You will get an API key.

    Copy the API key, as it is. You will need this in the following steps.

  4. Click Finish.

Configure server

  1. Download the Authy installer.
    curl "https://raw.githubusercontent.com/authy/authy-ssh/master/authy-ssh" -o authy-ssh
  2. Install Authy.
    sudo bash authy-ssh install /usr/local/bin

    It will prompt you to enter the API key that you received from the Authy web site. When prompted for what to do when Authy is down, choose option 1.

Configure two-factor for the user

  1. Run the following command to configure the user.
    sudo /usr/local/bin/authy-ssh enable user_name email_id country_code
  2. Open the /etc/ssh/sshd_config
    sudo vim /etc/ssh/sshd_config
  3. Go to the bottom of the file. Uncomment the ForceCommand line. In case the ForceCommand is not present, then manually enter it as the following:
    ForceCommand /usr/local/bin/authy-ssh login
  4. To save and exit press Esc and type the :wq!
  5. Restart the SSH server.
    sudo service sshd restart
  6. Make sure that the command is located in the your/etc/ssh/sshd_config file.
    cat /etc/ssh/sshd_config | grep ForceCommand
  7. Uninstall authy-ssh using the following command:
    sudo /usr/local/bin/authy-ssh uninstall
  8. Restart the ssh server for the changes to take effect.
    sudo service sshd restart
