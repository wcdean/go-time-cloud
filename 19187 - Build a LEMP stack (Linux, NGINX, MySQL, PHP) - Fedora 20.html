Install and configure a LEMP stack for a Fedora 20 server.

Build a LEMP stack (Linux, NGINX, MySQL, PHP) - Fedora 20

Difficulty: 3
Time: 20 minutes

LEMP is an unsual acronym because it really should be LNMP (Linux™, NGINX™, MySQL™, PHP™).  The "E" stands for NGINX (Engine X) and is probably used because of the wide acceptance of LAMP to refer to a stack built on the Apache operating system. Installing a LEMP stack creates a platform for your cloud server that includes:

  • An operating system (Linux)
  • A web server (NGINX)
  • A database (MySQL or Mariadb)
  • An application server (PHP)

You can build a fully functioning web site and/or use it as a platform for other applications.

MySQL (Mariadb)

  1. Get the local install for Fedora.
    sudo yum update
  2. Install MySQL (Mariadb).
    sudo yum install mysql-server

    There may be a prompt asking you if it is okay to install a key. Answer "y."

  3. Activate MySQL (Mariadb).
    sudo systemctl start mariadb
    sudo systemctl enable mariadb
  4. Secure your database server.
    sudo mysql_secure_installation

    The prompt will request the answer to a number of questions.

    The following table contains the remainder of the questions you will be asked, the recommended response, and some remarks regarding the consequences of not following the recommendation.

    Question Suggested Answer Why / Why not?
    Enter current password for root Press Enter for none. Do not enter a password. If you do so, MySQL will not have sufficient access to system resources to function as a database server.
    Do you want to change the root password? y A password is needed to secure MySQL. Enter a strong password and write it down.
    Remove anonymous users? y The anonymous user allows anyone to login without requiring an account created by an admin. A hacker attempting to compromise your database server may try logging in as this user.
    Disallow root login remotely? y It is more secure to create a non-root user with admin permissions. Answer "no," if you need more flexibility to manage your database server.
    Remove test database and access to it? y If you have a reason to keep a test database, answer "no."
    Reload privilege tables now? y Why wait?

  5. Activate MySQL.
    sudo systemctl restart mariadb

Install and Start NGINX

  1. Install NGINX.
    sudo yum install nginx
  2. Start the NGINX service.
    sudo systemctl start nginx

NGINX is now running as a web server. You can verify this using your browser to navigate to the IP address of your cloud server, e.g. http://IP Address of your cloudserver.

Install PHP

  1. Install PHP.
    sudo yum install php php-mysql php-fpm
  2. Configure the php.ini file.

    Open the /etc/php.ini file and find the cgi.fix_pathinfo directive. Most of the file is commented out and it will be about halfway through the file under a section called Paths and Directories. Uncomment the line and set it as follows:

    cgi.fix_pathinfo=0
  3. Save and close.
  4. Configure the www.conf listen directive.

    Open /etc/php-fpm.d/www.conf and find the listen directive and verify/edit it as follows:

    listen = /run/php-fpm/php-fpm.sock
  5. Find the listen.owner, listen.group, and listen.mode directives and modify them as follows: 
    listen.owner = nginx
listen.group = nginx
listen.mode  = 0666
  6. Save and close.
  7. Restart PHP.
    sudo systemctl restart php-fpm
  8. Change permissions and ownership on php-fpm.sock.
    sudo chmod 666 /run/php-fpm/php-fpm.sock
sudo chown nginx /run/php-fpm/php-fpm.sock

Configure NGINX

All of your LEMP stack components are now installed. The PHP/NGINX configuration in Fedora comes completely pre-configured, but the directives are buried in include files and obfuscated by multi-server proxy syntax. Even if are installing on Fedora, it may be instructive to read through the CentOS configuration steps, below.

  1. Open /etc/nginx/nginx.conf and verify/modify it such that it includes the following lines in the http context:
    index index.php index.html index.htm;
  2. In the same file, verify/modify it such that it includes the following lines in the server context:
    include /etc/nginx/default.d/*.conf;

    This line requests that any file with a conf extension be referenced at this point in the nginx.conf file. If the unlikely event this line does not exist, add it.

  3. Save and close.
  4. Navigate to /etc/nginx/default.d/php.conf and edit it. If this file does not exist, create it. In any event, edit the file so the PHP location context looks like the example below: 
                 # pass the PHP scripts to FastCGI server listening on the php-fpm socket
              location ~ \.php$ {
              try_files $uri =404;
              fastcgi_pass unix:/run/php-fpm/php-fpm.sock;
              fastcgi_index index.php;
              fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
              include fastcgi_params;
 }

    As long as the fastcgi_pass in the nginx.conf file matches the listen directive in the PHP configuration file, NGINX should be able to pass requests on to PHP.

  5. Save and close.
  6. Restart NGINX.
    sudo systemctl restart nginx
  7. If NGINX fails to restart, check your edits and make sure all of your braces ({}) are closed and that each directive ends with a semi-colon (;).

Verify PHP is running

  1. Create a test file /usr/share/nginx/html/info.php with the following contents:
    <?php
    phpinfo();
    ?>
  2. Open a browser and type: http://IP address or domain name of your server/info.php You will see a page full of statistics with a PHP logo in the upper right-hand corner.
