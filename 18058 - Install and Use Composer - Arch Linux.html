Install and use Composer

Install and Use Composer - Arch Linux

Difficulty: 2
Time: 10 minutes

Composer is a tool used to manage PHP program dependencies. You will learn how to install Composer on an Arch Linux system.

Install dependencies

  1. Install PHP and other required tools.
    sudo pacman -Syu php php-fpm php-mcrypt curl git
  2. Open your php.ini file for editing.
    sudo vim /etc/php/php.ini
    In the file, we'll be making a few changes.
  3. Find the cgi.fix_pathinfo directive, uncomment it and set it to zero.
    cgi.fix_pathinfo=0
  4. Uncomment these extensions as well:
    extension=phar.so
    extension=openssl.so
  5. Comment out the open_basedir directive.
    open_basedir=/srv/http/:/home/:/tmp/:/usr/share/pear/:/usr/share/webapps/
  6. Save and close.

Install Composer

  1. Run the Composer installation script.
    cd ~
    sudo curl -sS https://getcomposer.org/installer | php

    The script creates a file, composer.phar, which is a php archive file, in your project directory.

  2. Optionally, change the name of the newly created composer.phar file to composer.
    sudo mv composer.phar /usr/local/bin/composer
  3. Edit /etc/profile to change your path so that Composer will execute. Open the file to edit.
    sudo vim /etc/profile
  4. Add this code to the end of the file. 
    PATH=$PATH:/usr/local/bin/composer
export PATH

Using Composer

There are two ways you can use Composer: Create a file to manage your project or use the create-project command.

Create a composer.json file to manage your project

  1. Run this command:
    sudo vim composer.json
  2. Add the following contents to your file: 
    {
    "require": {
        "vendorname/projectname": "version [wildcards are supported]"
    }
}
    For example, use Laravel: 
    {
    "require": {
        "laravel/laravel": "4.2.*"
    }
}
  3. Run Composer to install dependencies for your Laravel project:
    composer install
    This may take a few minutes.

Use the create-project command

Because the ultimate goal when using Composer is to create a project, it may make sense to simply create the project using Composer's create-project command rather than configure dependencies.

This is the create-project syntax:
composer create-project [vendor]/[project] {target directory} [version]
For example, you can use the following to create a Laravel project:
composer create-project laravel/laravel my/project/dir 4.2
