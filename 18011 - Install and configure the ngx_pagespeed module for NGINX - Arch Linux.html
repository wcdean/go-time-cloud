Installation and configuration of Nginx with ngx_pagespeed

Install and configure the ngx_pagespeed module for Nginx - Arch Linux

Difficulty: 1
Time: 15 minutes

Nginx is fast. PageSpeed makes it faster by optimizing your website. This article describes how to install ngx_pagespeed, combine it with the Nginx source, and then compile and configure the resulting customized Nginx web server.

Prerequisites

  1. Update your server.
    sudo pacman -Syu
  2. Install a C compiler and some other tools.
    sudo pacman -S gcc make unzip
  3. Install perl.
    wget http://sourceforge.net/projects/pcre/files/latest/download?source=files

Install ngx_pagespeed

  1. Find the latest NPS version, and then use it to get the zip file from the archive. This article uses version 1.9.32.6.
    sudo wget https://github.com/pagespeed/ngx_pagespeed/archive/release-1.9.32.6-beta.zip
  2. Unzip the file.
    unzip release-1.9.32.6-beta.zip
  3. Navigate to the ngx_pagespeed folder.
    cd ngx_pagespeed-release-1.9.32.6-beta/
  4. Download psol from Google.
    sudo wget https://dl.google.com/dl/page-speed/psol/1.9.32.6.tar.gz
  5. Extract to psol.
    tar -xzvf 1.9.32.6.tar.gz

Download Nginx source

  1. Check http://nginx.org/en/download.html for the latest version. This article uses version 1.8.0.
  2. Navigate to your home directory.
    cd ~
  3. Download Nginx source.
    sudo wget http://nginx.org/download/nginx-1.8.0.tar.gz
  4. Unzip it.
    sudo tar -xvzf nginx-1.8.0.tar.gz

Build Nginx from source

  1. Navigate to Nginx source directory.
    cd nginx-1.8.0/
  2. Add the PageSpeed module to the Nginx configuration.
    sudo ./configure --add-module=$HOME/ngx_pagespeed-release-1.9.32.6-beta
  3. Create a make file.
    sudo make
  4. Install from your make file
    sudo make install
    .

Configure Nginx

  1. Open the configuration file for editing.
    sudo vim /usr/local/nginx/conf/nginx.conf
  2. Add the following lines to your http context: 
    # http context 
http {
...    
    # pagespeed
    pagespeed on;
    pagespeed DisableRewriteOnNoTransform off;
    pagespeed XHeaderValue "Powered By ngx_pagespeed";
    pagespeed FileCachePath "/var/cache/ngx_pagespeed/";
    pagespeed EnableFilters combine_css,combine_javascript;
...
  3. Modify the server and location contexts by commenting out the root and index directives in the location contexts, and then implementing them in the server context instead. Changed sections are underlined. 
    # http context 
   server {
        listen       80;
        server_name  localhost;
        root   /usr/local/nginx/html;
        index  index.html index.htm;

        #charset koi8-r;

        #access_log  logs/host.access.log  main;

        location / {
        # root   html;    
        # index  index.html index.htm;    
        }

        #error_page  404              /404.html;

        # redirect server error pages to the static page /50x.html
        #
        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
        # root   html;     
        }
 ...
  4. Verify the install by checking the Nginx version information.
    sudo /usr/local/nginx/sbin/nginx -V
  5. The response will look something like this: 
    nginx version: nginx/1.8.0
built by gcc 5.2.0 (GCC)
configure arguments: --add-module=/home/randomcode/ngx_pagespeed-release-1.9.32.6-beta
  6. Start the Nginx service.
    sudo /usr/local/nginx/sbin/nginx

Conclusion

Nginx is now running as a web server enhanced with PageSpeed. You can verify by using your browser to navigate to the IP address of your cloud server, http://IP Address of your cloud server.
