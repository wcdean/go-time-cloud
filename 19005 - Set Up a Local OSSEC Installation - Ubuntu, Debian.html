Setting up a local OSSEC, Monitoring

Set Up a Local OSSEC Installation - Debian, Ubuntu

Difficulty: 4
Time: 45 Minutes

OSSEC is an open-source, host-based intrusion detection system. It performs log integrity checks, analysis, rootkit detection, Windows registry monitoring, active response and time-based alerting. It can monitor one or more servers and can be configured for real-time alerts. This article explains the installation of a local OSSEC on Debian or Ubuntu.

Optionally if you want to send email from a local server, you need to install Postfix. For details, refer to Install and Configure Postfix - Debian or Install and Configure Postfix - Ubuntu.

You need sudo privileges to complete all of these steps.

Prepare the system

  1. Update the server.
    sudo apt-get update
  2. Install the packages.
    sudo apt-get install build-essential inotify-tools tar

Download OSSEC and verify the package

You need to download the OSSEC tarball and a file with its cryptographic checksum to make sure that you are installing valid software. Then you can generate the MD5 and SHA1 checksum and compare them with the checksum file.
  1. Download OSSEC.
    sudo wget -U ossec http://www.ossec.net/files/ossec-hids-2.8.1.tar.gz
  2. Download the checksum file.
    sudo wget -U ossec http://www.ossec.net/files/ossec-hids-2.8.1-checksum.txt
  3. Verify the md5sum of the compressed tarball. md5sum is a computer program that calculates and verifies 128-bit MD5 hashes. The MD5 hash (or checksum) functions as a compact digital fingerprint of a file.
    md5sum -c ossec-hids-2.8.1-checksum.txt
    The output is as follows:
    ossec-hids-2.8.1.tar.gz: OK
    md5sum: WARNING: 1 line is improperly formatted
    You can ignore the warnings here. "OK" confirms that the file is good.
  4. Verify the SHA1 checksum. SHA1 produces a 160-bit output called a message digest. The message digest can be an input to a signature algorithm that generates or verifies the signature for the message. US Secure Hash Algorithm 1 is used by SHA1.
    sha1sum -c ossec-hids-2.8.1-checksum.txt
    The output is as follows:
    ossec-hids-2.8.1.tar.gz: OK
    sha1sum: WARNING: 1 line is improperly formatted
    You can ignore the warnings here. "OK" confirms that the file is good.

Find your SMTP server

During installation, OSSEC asks for your SMTP server. To determine the correct SMTP server, follow these steps.

  1. Use the dig command to query your provider's email exchange resource record.
    dig -t mx coolexample.com
    Many sections display, but we're interested only in ANSWER SECTION.
    For example, if you run the command using fastmail.com:
    dig -t mx fastmail.com
    The valid SMTP servers for this provider are shown under ANSWER SECTION. 
    ;; ANSWER SECTION:
fastmail.com.           3600    IN      MX      10 in1-smtp.messagingengine.com.
fastmail.com.           3600    IN      MX      20 in2-smtp.messagingengine.com.
    In this example, you can use in1-smtp.messagingengine.com or in2-smtp.messagingengine.com as the SMTP server.
  2. Make a note of any of these SMTP servers. You'll need it during the installation.

Install OSSEC

You can install OSSEC in many modes including server, agent, local, or hybrid mode. We'll install it in local mode.

  1. Expand the file.
    sudo tar -zxf ossec-hids-2.8.1.tar.gz
  2. Change the directory to the expanded one.
    cd ossec-hids-2.8.1
  3. Initiate the installation.
    sudo ./install.sh
  4. When prompted, choose a language or use the default by pressing Enter.

Verify the OSSEC email settings

You need to verify that the email credentials you provided and the email credentials that OSSEC autoconfigured are correct. Email settings are in ossec.conf, which is located in /var/ossec/etc.

  1. Open /var/ossec/etc/ossec.conf in a text editor.

    sudo vim /var/ossec/etc/ossec.conf

    Email settings are at the top of the file. The following describes each of these fields:

    • <email_to> is the email address provided during installation.
    • <email_from> is the email address from which the emails appear to be sent.
    • <smtp_server> is the SMTP server provided during setup.
  2. Update these email settings as needed. After the updates, the final section looks like this:
    <global>
    <email_notification>yes</email_notification>
    <email_to>email@coolexample.com</email_to>
    <smtp_server>mail.coolexample.com.</smtp_server>
    <email_from>ossec@ossec_server</email_from>
    </global>
  3. Save and close the file by pressing Esc and typing :wq! followed by pressing the Enter key.
  4. Start the server.
    sudo /var/ossec/bin/ossec-control start
    You'll receive an email message saying that OSSEC has started.

(Optional) Update the main configuration file

The main configuration file is /var/ossec/etc/ossec.conf. This file is used to configure OSSEC for additional alerts, number of emails, directories to be monitored, and so on.
  1. Open /var/ossec/etc/ossec.conf in a text editor.
    sudo vim /var/ossec/etc/ossec.conf
  2. Add the following line in the <global> tag.
    <email_maxperhour>N</email_maxperhour>
    The final code block looks like this:
    <global>
    <email_notification>yes</email_notification>
    <email_to>email@coolexample.com</email_to>
    <smtp_server>mail.coolexample.com.</smtp_server>
    <email_from>ossec@ossec_server</email_from>
    <email_maxperhour>N</email_maxperhour>
    </global>
  3. Locate the <syscheck> tag in the same file. You can update the frequency of the system check by updating the <frequency> tag here. Time duration is in seconds.
    <syscheck>
    <!-- Frequency that syscheck is executed - default to every 22 hours -->
    <frequency>79200</frequency>
  4. Add the following line under the <syscheck> tag to add an alert for new files:
    <syscheck>
    <!-- Frequency that syscheck is executed - default to every 22 hours -->
    <frequency>79200</frequency>
    <alert_new_files>yes</alert_new_files>
  5. To enable realtime monitoring, locate the <directories> tag after the <syscheck> tag, and add report_changes="yes" realtime="yes" to each line:
    <!-- Directories to check (perform all possible verifications) -->
    <directories report_changes="yes" realtime="yes" check_all="yes">/etc,/usr/bin,/usr/sbin</directories>
    <directories report_changes="yes" realtime="yes" check_all="yes">/bin,/sbin</directories>
  6. Add the following line to add /home/user_name and /var/www to directories to be monitored:
    <directories report_changes="yes" realtime="yes" restrict=".php|.js|.py|.sh|.html" check_all="yes">/home/user_name,/var/www</directories>
    The restrict setting says to monitor only specified formats.
  7. Save and close the file by pressing Esc and typing :wq! followed by pressing the Enter key.

(Optional) Add local rules

There are several files in the /var/ossec/rules directory to manage custom and default rules. The file ossec_rules.xml contains OSSEC's default rule definitions, and custom rules are defined in the local_rules.xml file. It's useful to review the default rule definitions in the ossec_rules.xml file. To create custom rules, copy and paste rules from the ossec_rules.xml file to the local_rules.xml file, and modify the rules as desired.
  1. Open /var/ossec/rules/ossec_rules.xml in any text editor.
    sudo vim /var/ossec/rules/ossec_rules.xml
  2. Locate rule 554 and copy it.
    Rule 554 looks like this:
    <rule id="554" level="0" overwrite="yes">
    <category>ossec</category>
    <decoded_as>syscheck_new_entry</decoded_as>
    <description>File added to the system.</description>
    <group>syscheck,</group>
    </rule>
  3. Close the file by pressing Esc, and typing :q! followed by pressing the Enter key.
  4. Open /var/ossec/rules/local_rules.xml in a text editor.
    sudo vim /var/ossec/rules/local_rules.xml
  5. Add the rule 554 you copied from ossec_rules.xml, and change the level to equal 7 as shown.
    <rule id="554" level="7" overwrite="yes">
    <category>ossec</category>
    <decoded_as>syscheck_new_entry</decoded_as>
    <description>File added to the system.</description>
    <group>syscheck,</group>
    </rule>
  6. Save and close the file by pressing Esc and typing :wq! followed by pressing the Enter key.
  7. Restart OSSEC.
    sudo /var/ossec/bin/ossec-control restart

    You'll receive an email when OSSEC restarts.

Trigger file changes

You'll receive alerts whenever you add, update, or delete any file in the directories monitored by OSSEC. Keep in mind that real-time notifications are sent only for the modification and the deletion of files and not for the addition of files. Alerts for file addition are sent based on every full system check. The frequency of the full system check is defined in the ossec.conf file.

Next steps

After following all these steps, you'll be able to set up a local OSSEC installation. OSSEC can be configured to one or more servers, called agents. To explore how to monitor an OSSEC agent using an OSSEC server, refer to Monitor OSSEC Agents Using an OSSEC Server - Ubuntu, Debian.
