Install and use Virtualmin - ALL DISTROS

Difficulty: Value from spreadsheet
Time: Time value

Webmin is a GUI tool that helps you manage the software packages installed on your server. Virtualmin is an extension of Webmin that extends its functionality to manage Virtual Private Servers (VPS).  You should already have Webmin installed if not, use one of the links below depending on your distribution of linux:

  • Arch Linux
  • Centos, Fedora
  • Debian, Ubuntu

Install Virtualmin

  1. Download the Virtualmin software.
    sudo wget http://software.virtualmin.com/gpl/scripts/install.sh -O /root/virtualmin-install.sh
    Your screen will display something like this:
    2015-09-03 13:32:53 (710 KB/s) - ‘/root/virtualmin-install.sh’ saved [47633/47633]
  2. Install Virtualmin.
    sudo sh /root/virtualmin-install.sh
  3. You may have logged into your server using an IP address rather than a fully qualified domain name (FQDN). The script may prompt you for your FQDN. If this is the case, provide it.
  4. Log in to your server using your FQDN.  Do not use your IP address.  https://coolexample.com:10000
  5. If prompted, provide your Webmin credentials to login.

Review the Virtualmin and Webmin UI

  • On the left hand side of the page, there is a menu. Click on the name to toggle the UI between Virtualmin and Webmin.

You will notice some differences from the standard Webmin interface, because the two tools offer slightly different functionality. Webmin manages your physical server, while Virtualmin manages the individual VPS that you have installed on your physical server.

Next steps

Use the post-installation wizard to familiarize yourself with Virtualmin, and use it to help you build out VPS and the software running on them.
