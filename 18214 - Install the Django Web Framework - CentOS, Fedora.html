Install the Django Web Framework - Arch Linux

Difficulty: 1
Time: 1.50

Django is a powerful framework for writing Python web applications. Frameworks let you focus on the unique portions of your application and let the tools do the heavy lifting. In this article, we'll demonstrate various methods of installing Django on an Arch Linux server.

Installing Django

There are a number of different ways in which to install Django, depending upon your needs and how you want to configure your development environment. Each has different advantages, and one method may lend itself better to your specific situation than others.
Here are some of the methods:

  • Global Install from Packages: Before installing any of the programs below, you should update the package manager: sudo pacman -Syu.
  • Global Install through pip: The pip tool is a package manager for Python packages. If you install pip, you can easily install Django on the system level for use by any user.
  • Install through pip in a Virtualenv: The Python Virtualenv package lets you create self-contained environments for various projects.
  • Development Version Install through git: If you wish to install the latest development version instead of the stable release, you'll have to acquire the code from the git repo.

With the above caveats and qualities in mind, select the installation method that best suits your needs from the instructions below.

Global Install from Packages

If you wish to install Django using the EPEL repositories, the process is very straightforward.

  1. Use the Pacman package manager to download and install the relevant packages.
    sudo pacman -S python-django
  2. Once you have pip, install Django globally.
    sudo pip install django

Global Install through pip

  1. Install pip from the EPEL repositories.
    sudo pacman -S python-pip
  2. Once you have pip, install Django globally.
    sudo pip install django
  3. Verify that the installation was successful.
    django-admin --version

As you can see, the version available through pip is more up-to-date than the one from the EPEL repository (Yours will likely be different from the above).

Install through pip in a Virtualenv

This allows you to select Python packages on a per-project basis regardless of conflicts with another project's requirements.

  1. Install pip from the EPEL repository.
    sudo Pacman -s install python-pip
  2. Once pip is installed, use it to install the virtualenv package.
    sudo pip install virtualenv
  3. Now, whenever you start a new project, you can create a virtual environment for it. Start by creating and moving into a new project directory.
    mkdir ~/newproject
    cd ~/newproject
  4. Create a virtual environment within the project directory.
    virtualenv newenv

    This will install a standalone version of Python, as well as pip, into an isolated directory structure within your project directory.

  5. To install packages into the isolated environment, you must activate it.
    source newenv/bin/activate
    Your prompt should change to reflect that you are now in your virtual environment. It will look something like this:
    (newenv)username@hostname:~/newproject$.
  6. In your new environment, use pip to install Django. Note that you do not need to use sudo since you are installing locally.
    pip install django
  7. Verify the installation.
    django-admin --version
    1.7.5
  8. To leave your virtual environment, issue the deactivate command from anywhere on the system.
    cd ~/newproject
    source newenv/bin/activate

Install through git

If you need a development version of Django, you'll have to download and install Django from its git repository.

  1. Install git on your system with Pacman. We'll also install the pip Python package manager. We'll use this to handle the installation of Django after it has been downloaded.
    sudo Pacman -s install git python-pip
  2. Once you have git, you can clone the Django repository. You can clone the repository to a directory called django-dev within your home directory by typing:
    git clone git://github.com/django/django ~/django-dev
  3. Once the repository is cloned, you can install it using pip. We'll use the -e option to install in "editable" mode, which is needed when installing from version control.
    sudo pip install -e ~/django-dev
  4. Verify that the installation was successful.
    django-admin --version 1.9.dev20150305200340

    You can also combine this strategy with the use of virtualenv above if you wish to install a development version of Django in a single environment.

Creating a project

Now that you have Django installed, we can show you briefly how to get started on a project.

  1. Use the django-admin command to create a project.
    django-admin startproject projectname
    cd projectname
    This creates a directory called projectname within your current directory. Within this, a management script will be created, and another directory called projectname will be created with the actual code.

    If you were already in a project directory that you created for use with the virtualenv command, you can tell Django to place the management script and inner directory into the current directory without the extra layer by typing this (notice the ending dot): django-admin startproject projectname.

  2. Bootstrap the database (This uses SQLite by default) on more recent versions of Django.
    python manage.py migrate
  3. If the migrate command doesn't work, you likely are using an older version of Django. Instead, you can run the following:
    python manage.py syncdb
  4. You'll be asked to create an administrative user as part of this process. Select a user name, email address, and password for the user. If you used the migrate command above, you'll need to create the administrative user manually. You can create an administrative user by running this command:
    python manage.py createsuperuser
  5. You'll be prompted for a user name, an email address, and a password for the user. Once you have a user, you can start up the Django development server to see what a fresh Django project looks like. You should use this only for development purposes. Run this command:
    python manage.py runserver 0.0.0.0:8000
  6. Visit your server's IP address followed by :8000 in your web browser.
    server_ip_address:8000
  7. Aappend /admin to the end of your URL to get to the admin login page.
    server_ip_address:8000/admin

Conclusion

Congratulations! You have just learned how to set up Django Framework on Arch Linux.
