Install and configure a LEMP stack on Fedora 21 server.

Build a LEMP stack (Linux, NGINX, MySQL, PHP) - Fedora 21

Difficulty: 3
Time: 20 minutes

To build a dynamic web application, you need what has been coined a “stack” – which is developer lingo for an integrated set of software that has all of the components your application needs.

Most stacks have the same types of component, and differ mainly in which pieces of software they use for those components. For example, we’ll look at LEMP (which this article covers):

Component LAMP Software
Operating system Linux (e.g. Fedora)
Web server NGINX (pronounced engine-x)
Database MySQL
Programming language PHP

If you're astute, maybe you noticed that LEMP is basically an acronym for the software the stack uses (LNMP is difficult to pronounce in English, so they used “E” for NGINX because of its pronunciation).

In this guide, we'll walk you through installing all of these compontents (except for Linux, which is already installed as your OS when you create the server).

Install MySQL

  1. Get the local install for Fedora:
    sudo yum localinstall http://dev.mysql.com/get/mysql-community-release-fc21-6.noarch.rpm
  2. Install MySQL:
    sudo yum install mysql-community-server
    There might be a prompt asking you if it is okay to install a key. Answer y.
  3. Activate MySQL:
    sudo systemctl start mysql.service
  4. Secure your database server:
    sudo /usr/bin/mysql_secure_installation
    The prompt will request the answer to a number of questions:
    Question Suggested Answer Reasoning
    Enter current password for root Press return/enter for none. Do not enter a password. If you do so, MySQL will not have sufficient access to system resources to function as a database server.
    Do you want to change the root password? y A password is needed to secure MySQL. Enter a strong password and write it down.
    Remove anonymous users? y The anonymous user allows anyone to login without requiring an account created by an admin. A hacker attempting to compromise your database server may try logging in as this user.
    Disallow root login remotely? y It is more secure to create a non-root user with admin permissions. Answer "no," if you need more flexibility to manage your database server.
    Remove test database and access to it? y If you have a reason to keep a test database, answer "no."
    Reload privilege tables now? y Why wait?
  5. Activate MySQL:
    sudo systemctl restart mysql.service

Install and start NGINX

  1. Install NGINX:
    sudo yum install nginx
  2. Start the NGINX service:
    sudo systemctl start nginx.service
  3. To verify that NGINX is installed, type this URL in a browser:
    http://your server's IP address

    COMPANY_NAME Cloud Server customers can use

    .

If NGINX is installed, a Welcome to nginx! page displays.

Install and configure PHP

Install PHP

  • Install PHP:
    sudo yum install php php-mysql php-fpm

Configure the php.ini file

  1. Open your php.ini file:
    sudo vim /etc/php.ini
  2. Find the cgi.fix_pathinfo directive. Most of the file is commented out and it will be about halfway through the file, under a section called Paths and Directories. Uncomment the line and set it as follows:
    cgi.fix_pathinfo=0
  3. Save and close the file:
    :wq!

Configure the www.conf file

  1. Open your /etc/php-fpm.d/www.conf file:
    sudo vim /etc/php-fpm.d/www.conf
  2. Find the listen directive and ensure it looks like this:
    listen = /run/php-fpm/php-fpm.sock
  3. Find the owner and group and ensure they look like this:
    user = nginx
    group = nginx
  4. Find the listen.owner, listen.group, and listen.mode and ensure they look like this:
    listen.owner = nginx
    listen.group = nginx
    listen.mode = 0666
  5. Save and close the file:
    :wq!
  6. Restart PHP:
    sudo systemctl restart php-fpm

Change permissions and ownership on php-fpm.sock

  1. Navigate to the /run/php-fpm directory:
    cd /run/php-fpm
  2. Modify the permissions and ownership for the php-fpm.sock file as follows:
    sudo chmod 666 php-fpm.sock
    sudo chown nginx php-fpm.sock
  3. Restart PHP:
    sudo systemctl restart php-fpm

Configure NGINX

All of your LEMP stack components are now installed. The PHP and NGINIX configuration in Fedora comes completely pre-configured, but the directives are buried in include files and obfuscated by multi-server proxy syntax. Ensuring it's properly configured is a good idea.

  1. Open /etc/nginx/nginx.conf:
    sudo vim /etc/nginx/nginx.conf
  2. Find the the http context, and ensure the line beginning with index looks like this:
    index index.php index.html index.htm;
  3. Find the server context and make sure it contains an uncommented line that looks like this:
    include /etc/nginx/default.d/*.conf;
  4. Save and close the file:
    :wq!
  5. Open /etc/nginx/default.d/php.conf:
    sudo vim /etc/nginx/default.d/php.conf
  6. Either uncomment out or add the following lines in the php location context:
    # pass the PHP scripts to FastCGI server listening on the php-fpm socket

    location ~ \.php$ {
                       try_files $uri =404;
                       fastcgi_pass unix:/run/php-fpm/php-fpm.sock;
                       fastcgi_index index.php;
                       fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
                       include fastcgi_params;
                      } 

    As long as the fastcgi_pass in the nginx.conf file matches the listen directive in the PHP configuration file, NGINX should be able to pass requests on to PHP.

  7. Save and close the file:
    :wq!
  8. Restart NGINX:
    sudo systemctl restart nginx

If NGINX fails to restart, check your edits and make sure all of your braces ({}) are closed, and that each directive ends with a semi-colon (;).

Verify PHP is running

  1. Create a info.php file to ensure PHP is running:
    sudo vim usr/share/nginx/html/info.php
  2. Paste in the following contents:
    <?php
    phpinfo();
    ?>
  3. Go to:
    http://your server's IP address/info.php

A page full of statistics with a PHP logo in the upper right-hand corner will be displayed.
