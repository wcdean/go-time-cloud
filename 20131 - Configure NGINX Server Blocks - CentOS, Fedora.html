Configure NGINX Server Blocks - CentOS


Difficulty: 2
Time: Time value

NGINX has rapidly become one of the most popular web server solutions in the industry. Considered to be a competitor to the traditional Apache web server, NGINX is known for being faster and easier to scale than other web servers.

Unlike Apache, which uses Virtual Hosts to manage individual sites and domains, NGINX uses server blocks. Server blocks let you host multiple sites with different domains on the same server. This guide will cover how to set up server blocks for your domains and subdomains. These guide focuses on setting up NGINX on CentOS.

Prerequisites

  1. You must have an CentOS 6 or 7 server with NGINX installed. If you do not have NGINX installed, follow the NGINX install steps in the LEMP stack guide for CentOS 6 or CentOS 7.
  2. Log in to your server with a user that has sudo privileges. If you have a GoDaddy Cloud server, you will already have one by default.
  3. Point at least one domain or subdomain to your server. You will use these for your NGINX server blocks.
  4. Alternatively, if you don't have a domain you want to use yet, you can simply use your Server IP address for the configuration, or edit your hosts file to test the configuration.

This guide will demonstrate setting up one domain and one subdomain on separate NGINX server blocks.

Create directory structures and permissions

CentOS uses /usr/share/nginx/html as the base root for the NGINX server overall. You will create separate directories in the /usr/share/nginx directory for each of your server blocks.

  1. Create a directory for your first domain or subdomain.
    sudo mkdir -p /usr/share/nginx/site1.coolexample.com
  2. Give this directory the correct permissions ownership for the NGINX user.
    sudo chown -R nginx:nginx /usr/share/nginx/site1.coolexample.com
    sudo chmod -R 755 /usr/share/nginx/site1.coolexample.com
  3. Create a directory for your second domain or subdomain.
    sudo mkdir -p /usr/share/nginx/site2.coolexample.com
  4. Give this directory the correct permissions and ownership as well
    sudo chown -R nginx:nginx /usr/share/nginx/site2.coolexample.com
    sudo chmod -R 755 /usr/share/nginx/site2.coolexample.com

You now should have two directories, one for each site.

Create test index pages

In order to test your configuration, you want to create a few basic test index files. These will eventually be replaced with your actual website files, but for now this will demonstrate how NGINX server blocks differentiate site by directory.

  1. Create your first site's test file with a text editor.
    sudo vim /usr/share/nginx/site1.coolexample.com/index.html
  2. Paste this example HTML file in the editor. You can use your own text and domain as you see fit.
    <html> <head> <title>site1.coolexample.com</title> </head> <body> <h1>Hooray! NGINX is properly serving on <em>site1.coolexample.com</em>!</h1> </body> </html>
  3. Save and close the file
  4. Now do the same for the second site. Create the file:
    sudo vim /usr/share/nginx/site2.coolexample.com/index.html
  5. Then paste the example HTML content.
    <html> <head> <title>site2.coolexample.com</title> </head> <body> <h1>Hooray! NGINX is properly serving on <em>site2.coolexample.com</em>!</h1> </body> </html>
  6. Save and close the file.

Create the NGINX server blocks

Now for the important part. You will create the files and directoreis for the NGINX server block information.

Set up the server block directory tree

  1. To make your server blocks easy to enable and disable, create the following directories:
    sudo mkdir /etc/nginx/sites-enabled
    sudo mkdir /etc/nginx/sites-available
    This sort of directory system is typically used by Debian and Ubuntu, but they can be used by any Linux distribution running NGINX.
  2. You do need to add another line to your NGINX configuration file, so go ahead and open that in a text editor.
    sudo vim /etc/nginx/nginx.conf
  3. Look for the following include line:
    include /etc/nginx/conf.d/*.conf;
  4. Directly under this line, add the following line:
    include /etc/nginx/sites-enabled/*;
  5. Save and close the file.

Set up NGINX server blocks

  1. First, create a new file to use as a template. You will use this template for creating future NGINX server blocks.
    sudo vim /etc/nginx/sites-available/template
  2. Inside this file, paste the following content. This will give you the defaults that you need for the NGINX server block locations, and also accounting for working with PHP and FastCGI. 
              server {
              listen       80;
              root /usr/share/nginx/yoursitehere;
              index index.php index.html index.html;
              server_name yoursitehere;
            location / {
              root   /usr/share/nginx/yoursitehere;
              try_files $uri $uri/ /index.html;
                }
            location ~ \.php$ {
                     fastcgi_pass unix:/var/run/php-fpm/php-fpm.sock;
                     fastcgi_index index.php;
                     fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
                     include fastcgi_params;
                     }
            error_page  404              /404.html;
            location = /404.html {
              root   /usr/share/nginx/yoursitehere;
            }
            error_page   500 502 503 504  /50x.html;
            location = /50x.html {
            root   /usr/share/nginx/yoursitehere;
            }
          }
  3. Save and close the file.
  4. You now have a template file you can use for creating future server blocks. Copy these for each server block you create.
    sudo cp /etc/nginx/sites-available/template /etc/nginx/sites-available/site1.coolexample.com
    sudo cp /etc/nginx/sites-available/template /etc/nginx/sites-available/site2.coolexample.com
  5. In each of these new files, you need to replace yoursitehere with the domain you are setting up. There are a few ways to do this:
    • Open the file in a text editor, and manually edit the paths. This is useful if you have to make additional edits to the configuration file.
      sudo vim /etc/nginx/sites-available/site1.coolexample.com
      sudo vim /etc/nginx/sites-available/site2.coolexample.com
    • Use this sed command. This will automatically replace all instances of yoursitehere with the domain of your choice inside the configuration file.
      sudo sed -i 's/yoursitehere/site1.coolexample.com/g' /etc/nginx/sites-available/site1.coolexample.com
      sudo sed -i 's/yoursitehere/site2.coolexample.com/g' /etc/nginx/sites-available/site2.coolexample.com
  6. Add symbolic links in the sites-enabled directory to enable these server blocks.
    sudo ln -s /etc/nginx/sites-available/site1.coolexample.com /etc/nginx/sites-enabled/site1.coolexample.com
    sudo ln -s /etc/nginx/sites-available/site2.coolexample.com /etc/nginx/sites-enabled/site2.coolexample.com
  7. Restart NGINX to confirm all your changes.
    sudo service nginx restart

    If you get an error when restarting NGINX, it is likely that you have not properly changed the server block files in the sites-available directory. Make sure that you have all your { } brackets closed and that all directives end in a semicolon.

You should now be able to navigate to your site(s) in a browser, and see the NGINX test page that you created. If you did it correctly, each different site should display a different page.

Conclusion

You are now ready to upload your site or install your application to your new site directory, and get your site up and running on NGINX. For additional NGINX server configuration options, check out the Official NGINX documentation portal.
