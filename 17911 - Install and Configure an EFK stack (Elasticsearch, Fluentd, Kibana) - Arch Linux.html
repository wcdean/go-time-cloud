Install and Configure an EFK Stack (Elasticsearch, Fluentd, Kibana) - Arch Linux

Difficulty: 2
Time: 30 minutes

Using Elasticsearch, Fluentd, and Kibana (also known as an EFK stack) is a common way to perform data analysis. This combination lets you collect, index, search, and visualize log data. In this article, you'll learn how to install an EFK stack on Arch Linux.

This article assumes that you have completed the initial server setup for your host machine and that you have sudo user privileges.

Install and configure ElasticSearch

Install ElasticSearch

  1. Update pacman.
    sudo pacman -Syu
  2. Install the required packages.
    sudo pacman -S zip unzip unrar
  3. Install Java.
    sudo pacman -S jre8-openjdk
    To confirm a successful installation, check the Java version.
    java -version
  4. Install the Elasticsearch deb packages. These commands are for version 1.7.1. To obtain the latest version, visit the Elasticsearch download page.
    sudo wget https://download.elastic.co/elasticsearch/elasticsearch/elasticsearch-1.7.1.tar.gz
    sudo tar xzf elasticsearch-1.7.1.tar.gz
    sudo mv elasticsearch-1.7.1 /usr/share/elasticsearch

Configure ElasticSearch

  1. Open the Elasticsearch configuration file elasticsearch.yml. To make Elasticsearch accessible from the public Internet, you need to disable its dynamic scripting so that it can be accessed by the Kibana dashboards.
    cd /usr/share/elasticsearch
    sudo vim config/elasticsearch.yml
  2. Set the variable script.disable_dynamic to true by appending the following code at the end of the file:
    script.disable_dynamic: true
  3. Save and exit the file.
  4. Start Elasticsearch.
    sudo ./bin/elasticsearch
    A successful startup shows a started message.
    To stop the server, press Ctrl+C

Install and configure Kibana

Install Kibana

  1. Move to the home directory for Kibana installation.
    cd ~
  2. Install Kibana. These commands are for version 4.1.2. To obtain the latest version, visit the Kibana download page.
    sudo curl -L https://download.elastic.co/kibana/kibana/kibana-4.1.2-linux-x64.tar.gz | tar xzf -
    sudo cp -r kibana-4.1.2* /usr/share/kibana-4.1.2

Configure Kibana

  1. Open Kibana's configuration file to change the port from 9200 (the default) to 80, which is used for interaction with Elasticsearch.
    sudo vim /usr/share/kibana-4.1.2/config/kibana.yml
  2. Locate the following code, which is related to the port setting:
    elasticsearch_url: "http://localhost:9200"
  3. Update the content as follows:
    elasticsearch_url: "http://localhost:80"
  4. Save and exit the file.

Install and configure Nginx

  1. Install Nginx, which can be used as the proxy server. This enables access to the Kibana dashboards from the Internet.
    sudo pacman -S nginx
  2. Create and open the Nginx configuration file.
    sudo vim /etc/nginx/nginx.conf
  3. In this file, first make these changes: 
    #user html;
worker_processes  1;
 
#error_log  logs/error.log;
#error_log  logs/error.log  notice;
#error_log  logs/error.log  info;
 
...
   
    #gzip  on;
   
    server {
          listen       80;
          server_name  localhost;
          #charset koi8-r;
          access_log  /var/log/nginx/kibana.log;
   
          location /
			{
				root  /usr/share/kibana-4.1.2;
				index index.php index.html  index.htm;
			}
   ...
  4. Then, add this text in after that first location block, making sure to replace your own IP for your public IP. 
      location /
			{
				root  /usr/share/kibana-4.1.1;
				index  index.html  index.htm;
			}
  # Beginning of new contents - added below:

  location ~ ^/_aliases$
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
			}
   
		  location ~ ^/.*/_aliases$
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
			}
   
		  location ~ ^/_nodes$
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
			}
   
		  location ~ ^/.*/_search$
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
			}
   
		  location ~ ^/.*/_mapping
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
			}
   
		  # Password protected end points
		  location ~ ^/kibana-int/dashboard/.*$
			{
				proxy_pass http://your public IP:9200;
				proxy_read_timeout 90;
				limit_except
				GET
				{
					proxy_pass http://your public IP:9200;
					auth_basic "Restricted";
					auth_basic_user_file /etc/nginx/conf.d/kibana.myhost.org.htpasswd;
				}
			}
   
		  location ~ ^/kibana-int/temp.*$
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
				limit_except GET
				{
					proxy_pass http://127.0.0.1:9200;
					auth_basic "Restricted";
					auth_basic_user_file /etc/nginx/conf.d/kibana.myhost.org.htpasswd;
				}
			}
  5. Save and exit the file.
  6. Restart Nginx.
    sudo systemctl restart nginx

Install and configure Fluentd

In this section you'll install Fluentd via Ruby. First, you need to install the dependencies, including RVM.

Install Fluentd dependencies

  1. Download the RVM signatures.
    command curl -sSL https://rvm.io/mpapis.asc | gpg2 --import -
  2. Install RVM.
    sudo curl -L get.rvm.io | bash -s stable
  3. Open the .bashrc file.
    sudo vim .bashrc
  4. Add the following line at the end of the file:
    [[ -s "$HOME/.rvm/scripts/rvm" ]] && source "$HOME/.rvm/scripts/rvm"
  5. Save and close.
  6. Close the session, and log in to apply the changes.
  7. Install the dependencies of RVM.
    rvm requirements
  8. Install other dependencies before installing Ruby.
    sudo pacman -S libyaml
    sudo pacman -S git
    gem install bundle
  9. Install Ruby with RVM.
    rvm install ruby

Install Fluentd

  1. Fetch the Fluentd source code.
    git clone https://github.com/fluent/fluentd.git
    cd fluentd
  2. Build and install Fluentd. These commands are for version 0.12.16. For other versions, see the Fluentd download page.
    gem install bundler
    bundle install
    bundle exec rake build
    gem install pkg/fluentd-0.12.16.gem

Configure Fluentd

  1. Set up Fluentd.
    fluentd --setup /etc/fluent
  2. Run an optional quick test of the Fluent installation.
    fluentd -s conf
    fluentd -c conf/fluent.conf &
    '{"json":"message"}' | fluent-cat debug.test
    If fluent is running, your output should resemble the following:
    2015-10-22 06:00:20 -0400 debug.test: {"json":"message"}
    If fluent is not running, the output will look something like this:
    connect failed: Connection refused - connect(2) for "127.0.0.1" port 24224"
    If that's the case, see a full list of fluent commands.
    fluentd -help
  3. Install additional plugins.
    gem install fluent-plugin-elasticsearch
    gem install fluent-plugin-record-reformer
  4. Update the fluent configuration file. Assuming you installed fluent in your home directory, use this command:
    sudo vim ~/fluentd/fluent.conf
    Make sure this is indeed the configuration file that opens, and not a blank file. If it's a blank file, exit out of this and go to the fluentd directory and find the fluent.conf file.
  5. Add the following lines at the top of the file: 
    <source>
    type syslog
    port 5140
    tag system
 </source>
 <match system.*.*>
    type record_reformer
    tag elasticsearch
    facility ${tag_parts[1]}
    severity ${tag_parts[2]}
 </match>
 <match elasticsearch>
    type copy
    <store>
       type stdout
    </store>
    <store>
         type elasticsearch
         logstash_format true
         flush_interval 5s #debug
    </store>
</match>
  6. Save and exit the file.
  7. Test Fluentd.
    echo '{"json":"message"}' | fluent-cat debug.test

Next steps

Congratulations! You have successfully installed an EFK stack (Elasticsearch, Fluentd, and Kibana).
