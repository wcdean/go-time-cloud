How to install the Django Web Framework on Debian, Ubuntu

Difficulty: 1
Time: 1.50 hours

Django is a powerful framework for writing Python web applications. Frameworks allow you to focus on the unique portions of your application, and let the tools do the heavy lifting. In this article, we will demonstrate various methods of installing Django on a Debian and Ubuntu server.

Select an installation method for Django

There are a number of different ways in which you can install Django depending upon your needs, and how you want to configure your development environment. These have different advantages, and one method may lend itself better to your specific situation than others.

Here are some of the methods:

Global Install from Packages

The EPEL repository contains Django packages that can be installed easily with the conventional apt-get package manager. The process is straightforward.

  1. Update your local package repository and then install the python-django package.
    sudo apt-get update
    sudo apt-get install python-django
  2. Verify the installation by checking the installed version.
    django-admin --version
    1.6.1

Global install through pip

The pip tool is a package manager for Python packages. If you install pip, you can easily install Django on the system level for use by any user.

First, we need to install the pip package manager.

  1. Install pip from the EPEL repositories after updating your local packages.
    sudo apt-get update
    sudo apt-get install python-pip
  2. After you have pip installed, install Django globally.
    sudo pip install django
  3. To verify that the installation was successful, type:
    django-admin --version
    1.7.5

In the example, the version available through pip is more up-to-date than the one from the EPEL repository. It is likely that your versions will be different from the ones above.

Install through pip in a Virtualenv

The Python Virtualenv package allows you to create self-contained environments for various projects. This installation method allows you to select Python packages on a per-project basis, regardless of conflicts with other project's requirements.

  1. Install pip from the EPEL repository:
    sudo apt-get update
    sudo apt-get install python-pip
  2. Use pip to install the Virtualenv package.
    sudo pip install virtualenv
  3. Now, whenever you start a new project, you can create a virtual environment for it. Start by creating and moving into a new project directory:
    mkdir ~/testproject
    cd ~/testproject
  4. Create a virtual environment within the project directory by typing:
    virtualenv testenv

    This will install a standalone version of Python, as well as pip, into an isolated directory structure within your project directory.

  5. To install packages into the isolated environment, you must activate it by typing:
    source testenv/bin/activate
  6. Your prompt should change to reflect that you are now in your virtual environment. It will look something like this:
    (testenv)username@hostname:~/testproject$
  7. In your new environment, you can use pip to install Django. Note that you do not need to use sudo, because you are installing locally.
    pip install django
  8. Verify the installation.
    django-admin --version
    1.7.5
  9. Exit the virtual environment to return back to your prompt.
    deactivate
  10. You can re-activate your environment by going back to the project directory and using the activate command.
    cd ~/testproject
    source testenv/bin/activate

Development installation version through git

If you want to install the latest development version instead of the stable release, you will have to acquire the code from the git repo. For a development version of Django, download and install Django from its git repository.

  1. As always update the local repositories
    sudo apt-get update
  2. Install git on your system with apt-get. We will also install the pip Python package manager. We will use this to handle the installation of Django after it has been downloaded:
    sudo apt-get install git python-pip
  3. After you have git installed, you can clone the Django repository. Between releases, this repository will have up-to-date features and bug fixes that may affect stability. Clone the repository to a directory called django-dev within your home directory by typing:
    git clone git://github.com/django/django ~/django-dev
  4. After the repository is cloned, you can install it using pip. You will use the -e option to install in "editable" mode, which is needed when installing from version control:
    sudo pip install -e ~/django-dev
    To verify that the installation was successful by typing:
    django-admin --version
    1.9.dev20150305200340

    You can also combine this strategy with the use of virtualenv above, if you want to install a development version of Django in a single environment.

Create a sample project

Now that you have Django installed, we can briefly show you how to get started on a project.

  1. Use the startproject command to create a project :
    django-admin startproject projectname
    cd projectname
    This command creates a directory called projectname within your current directory. Within this, a management script will be created and another directory called projectname will be created with the actual code.

    If you were already in a project directory that you created for use with the virtualenv command, you can tell Django to place the management script and inner directory into the current directory without the extra layer by typing this command, including the ending period: django-admin startproject projectname.

  2. To bootstrap the database, which uses SQLite by default, on more recent versions of Django, you can type:
    python manage.py migrate
  3. If the migrate command doesn't work, it is likely that you are using an earlier version of Django. Instead, you can type:
    python manage.py syncdb
  4. As part of this process, create an administrative user. Select a username, email address, and password for the user.

    If you used the migrate command above, you'll need to create the administrative user manually. You can create an administrative user by typing:

    python manage.py createsuperuser
  5. You will be prompted for a username, an email address, and a password for the user.
  6. After you have a user set up, start up the Django development server to see what a fresh Django project looks like. You should only use this command for development purposes.
    python manage.py runserver 0.0.0.0:8000
  7. Now when you navigate to your web browser and type in http://your server IP:8000 and you will see a "It worked!" page.
  8. To go to the admin login page, append /admin to the end of your URL http://your server IP:8000/admin

Conclusion

Congratulations! You have just learned how to setup Django Framework on Debian and Ubuntu!
