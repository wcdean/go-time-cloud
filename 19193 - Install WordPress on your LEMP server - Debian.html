Installing Wordpress on a LEMP server for Debian.

Install WordPress on Your LEMP Server - Debian

Difficulty: 1
Time: 15 minutes

WordPress has become much more than its humble beginnings in 2003 as a self-hosted blogging tool. It's the Internet's leading content management system (CMS). This open-source CMS powers all types of sites, including many major brands. With a MySQL™, and PHP architecture, WordPress is easy to install and has endless options for website configuration thanks to tens of thousands of plugins and themes from its rich and active community.

This article walks you through setting up WordPress on a Debian system. The steps are for a non-root user with sudo, or administrative, privileges.

Prerequisites

Make sure you that have a non-root user with sudo (administrator) privileges set up. You also need a Linux™, Nginx™, MySQL, and PHP (LEMP) stack configured. Be sure that Nginx, MySQL, and PHP are up and running. As you install the software, make a note of your MySQL root account (administrator) password. You'll need it for these tasks.

Create a MySQL database and administrator for WordPress

Set up your MySQL database with an administrator so that WordPress has a place to store all your site and user information.

  1. Log into your MySQL admin account.
    sudo mysql -u root -p

    Remember that all MySQL code statements must end with a semi-colon (;).

  2. At the prompt, enter your MySQL root password.
  3. At the MySQL prompt, create your WordPress database. We'll use the database name "wordpress."
    CREATE DATABASE wordpress;
  4. Create the database administrator account, and set the user's credentials. We'll use "wordpressadmin" as the example user name and "wordpresspassword" as the password.
    CREATE USER wordpressadmin@localhost IDENTIFIED BY 'wordpresspassword';

    Make sure to create a strong password for best security practices. Keep your user name and password handy, as you'll need them later.

  5. Give, or grant, your admin permissions to the database.
    GRANT ALL PRIVILEGES ON wordpress.* TO wordpressadmin@localhost;
    GRANT is one of the account management statements that the MySQL server notices and loads into memory right away. So a manual reload/restart should not be necessary. But you can check the permissions using SHOW GRANTS.
    SHOW GRANTS FOR 'wordpressadmin'@'localhost';
  6. If the privileges don't take effect, flush the privileges to activate the user. FLUSH forces a reload of the cache.
    FLUSH PRIVILEGES;
  7. Exit MySQL to return to your normal command prompt.
    exit

You finished!

Download and install WordPress

Your system is now ready to install WordPress into your directory. WordPress is available in a tar.gz or zip format. We'll use the tar.gz file.
  1. Download the WordPress files from the official site using wget.
    cd ~
    wget http://wordpress.org/latest.tar.gz
    This URL always links to a compressed file of the latest stable version of WordPress.
  2. Extract, or untar, the application directory.
    tar xzvf latest.tar.gz
    This command creates a directory called wordpress in your home directory.

Configure WordPress

  1. Go into your WordPress directory.
    cd ~/wordpress
  2. Edit the following lines with the values in wp-config.php.
    sudo cp wp-config-sample.php wp-config.php
  3. Open the configuration file for editing.
    sudo vim wp-config.php
  4. Look for the following section of the code (starting with MySQL settings…), and replace the values of the three DB_ variables (database name, database username, mysql root password) with the credentials you just created.
    // ** MySQL settings - You can get this info from your web host ** //
    /** The name of the database for WordPress */
    define('DB_NAME', 'wordpress');
    /** MySQL database username */
    define('DB_USER', 'wordpressadmin');
    /** MySQL database password */
    define('DB_PASSWORD', 'wordpresspassword');
  5. Save and close the file.

Copy files to document root

  1. Create the directory.
    sudo mkdir -p /var/www/html
  2. Install rsync if it's not already installed. (We're going to use it to copy the files.)
    sudo apt-get install rsync
  3. Copy the files over using rsync.
    sudo rsync -avP ~/wordpress/ /var/www/html/

Create an uploads folder

  1. Switch back to the root directory before creating a new folder.
    cd /var/www/html/
  2. Create a directory to house WordPress content/uploads.
    mkdir wp-content/uploads
  3. Change ownership to www-data.
    sudo chown -R :www-data /var/www/html/wp-content/uploads

Update the Nginx configuration file

You now need to update the default Nginx configuration file with your WordPress settings. Use the default Nginx server block as a base for your new server block.

  1. First, create a backup of your config file.
    sudo cp /etc/nginx/sites-available/default /etc/nginx/sites-available/default.bak
  2. Now, make a copy of the config file to use for WordPress.
    sudo cp /etc/nginx/sites-available/default /etc/nginx/sites-available/wordpress
  3. Open the new file for editing.
    sudo vim /etc/nginx/sites-available/wordpress
  4. In the output file, make these changes:
    • Point root to /var/www/html: Find the line root /usr/share/nginx/html; and change /usr/share/nginx/html; to /var/www/html;.
    • Optionally, point server_name to your site's domain or IP. For example, the modified line may look like server_name coolexample.com;
    • Modify the try_files code to look like this:
      location / {
      # try_files $uri $uri/ =404;
      try_files $uri $uri/ /index.php?q=$uri&$args;
      }
      This enables WordPress pretty permalinks. If the entire try_files part is missing, paste this block after server name:
      location / {
      try_files $uri $uri/ /index.php?q=$uri&$args;
      }
    After the changes, your file resembles this: 
    server {
        listen 80 default_server;
        listen [::]:80 default_server ipv6only=on;

        root /var/www/html;
        index index.php index.html index.htm;

        server_name your IP or domain name;

        location / {
                # try_files $uri $uri/ =404;
                try_files $uri $uri/ /index.php?q=$uri&$args;
        }

        location ~ /.php$ {
                try_files $uri =404;
                fastcgi_split_path_info ^(.+\.php)(/.+)$;
                fastcgi_pass unix:/var/run/php5-fpm.sock;
                fastcgi_index index.php;
                include fastcgi_params;
        }
}
  5. Save and close the file.

Enable your WordPress site

Create a symlink to reference your new location, and delete the old file.
  1. Link your sites-enabled directory to this new file.
    sudo ln -s /etc/nginx/sites-available/wordpress /etc/nginx/sites-enabled/
  2. Disable Nginx's default site because it conflicts with the file we just linked. We'll do that using rm.
    sudo rm /etc/nginx/sites-enabled/default
  3. Restart Nginx and the PHP processes for your changes to take effect.
    sudo service nginx restart
    sudo service php5-fpm restart

Install WordPress

Good news! The rest of your installation uses the WordPress web interface.

  1. Enter your server’s domain name or public IP address in your web browser: http://your_domain_name_or_IP.
    You are redirected to the WordPress admin page: http://server_domain_name_or_IP/wp-admin/install.php
  2. If you get the preliminary page to choose your language, do so and select Continue.
  3. At the WordPress 5-minute installation page, fill out your information to create your administrator account. Then, click Install WordPress.

  4. On the next screen, WordPress confirms your installation with your user name and (hidden) password. Click Log In.

  5. WordPress then prompts you to log in again with your new credentials. Do so now and click Log In.

All done! You'll be taken to the dashboard of your new site.
