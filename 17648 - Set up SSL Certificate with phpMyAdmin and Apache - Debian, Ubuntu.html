This article describes the configuration steps required to make your phpMyAdmin implementation use SSL.

Set up a SSL Certificate with phpMyAdmin and Apache - Debian, Ubuntu

Difficulty: 2
Time: 20 minutes

phpMyAdmin is an open source tool that enables users to perform MySQL database operations like create, delete, or modify MySQL databases or execute SQL statements, by providing a user interface. An essential way to help safeguard your data is to encrypt traffic between the web server client and your phpMyAdmin implementation.

This article describes how to use SSL certificate to allow phpMyAdmin traffic to be encrypted.

Pre-requisites

You need to have the following installed on your server:
  • LAMP for Ubuntu or LAMP for Debian
  • phpMyAdmin on Ubuntu or phpMyAdmin on Debian
  • SSL Certificate

Force phpMyAdmin to your SSL server

After you have all the prerequisites in place, the only thing left to do is to force phpMyAdmin to use SSL instead of http.

  1. Open the phpMyAdmin configuration file.
    sudo vim /etc/phpmyadmin/config.inc.php
  2. Add the following line towards the end of the file:
    $cfg['ForceSSl'] = true;
  3. Save and exit the file.
  4. When you go to your browser, navigate to http://yourdomainname.com/phpmyadmin, and then enter your user credentials, you will see in the address bar that it uses https://.

Conclusion

Access to phpMyAdmin will now be limited to https://yourdomainname.com only.
